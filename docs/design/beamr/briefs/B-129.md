# B-129: erlang BIFs batch 4 — hashing, time, and misc

> **Cluster:** beamr
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

Remaining erlang module BIFs needed across OTP: phash2 (portable hash), unique_integer (monotonic unique values), monotonic_time (monotonic clock), system_time (wall clock), and miscellaneous utilities.

## Task

Register ~10 remaining utility BIFs.

## Requirements

### R1: Hashing BIFs

erlang:phash2/1 and erlang:phash2/2 — portable term hashing matching BEAM's algorithm.

**Acceptance:**
- phash2(Term) returns integer in 0..2^27-1
- phash2(Term, Range) returns integer in 0..Range-1
- Same term always produces same hash (deterministic)
- Hash is consistent with ETS term hashing from B-091

**Files:**
- modify: crates/beamr/src/native/gate3_bifs/mod.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R2: Time BIFs

erlang:monotonic_time/0, erlang:system_time/0, erlang:monotonic_time/1, erlang:system_time/1, erlang:time_offset/0.

**Acceptance:**
- monotonic_time() returns monotonically increasing native time units
- system_time() returns wall clock in native time units
- monotonic_time(millisecond) converts to milliseconds
- time_offset() returns the difference between system and monotonic time

**Files:**
- modify: crates/beamr/src/native/gate3_bifs/mod.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R3: Unique value BIFs

erlang:unique_integer/0, erlang:unique_integer/1 with options [positive, monotonic].

**Acceptance:**
- unique_integer() returns unique values across calls
- unique_integer([positive]) returns only positive values
- unique_integer([monotonic]) returns monotonically increasing values
- Thread-safe (uses AtomicU64)

**Files:**
- modify: crates/beamr/src/native/gate3_bifs/mod.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R4: Misc utility BIFs

erlang:min/2, erlang:max/2, erlang:abs/1, erlang:binary_to_term/1, erlang:term_to_binary/1.

**Acceptance:**
- min(1,2) returns 1, min(a,b) returns a (term ordering)
- max(1,2) returns 2
- abs(-5) returns 5, abs(5) returns 5
- binary_to_term/term_to_binary round-trip using ETF from B-117

**Files:**
- modify: crates/beamr/src/native/gate3_bifs/mod.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

- SHALL NOT implement os:timestamp — use monotonic_time
- binary_to_term/term_to_binary depend on B-117 ETF encoder/decoder

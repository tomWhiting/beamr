# B-123: Net_kernel and connection supervision — connect_node, nodes/0

> **Cluster:** beamr
> **Depends on:** B-119
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

net_kernel is the process that manages distribution connections. It provides connect_node/1 (establish connection to a node), disconnect_node/1 (tear down connection), and nodes/0 (list connected nodes). In beamr, net_kernel is implemented as BIFs backed by the ConnectionManager rather than as a BEAM process.

## Task

Implement net_kernel BIFs for connection management.

## Requirements

### R1: net_kernel:connect_node/1

Connect to a named node. Resolves the name, establishes TCP connection, performs handshake.

**Acceptance:**
- connect_node(NodeName) returns true on successful connection
- Returns false if connection fails (name not resolved, handshake rejected)
- If already connected, returns true (idempotent)
- Test: connect two nodes, verify nodes() lists both

**Files:**
- modify: crates/beamr/src/distribution/mod.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R2: erlang:nodes/0 and erlang:disconnect_node/1

List connected nodes and disconnect from a specific node.

**Acceptance:**
- nodes() returns list of connected node name atoms
- disconnect_node(NodeName) closes the connection and returns true
- After disconnect, nodes() no longer lists that node
- Disconnection triggers noconnection exit signals (B-122 R3)
- Test: connect, verify in nodes(), disconnect, verify removed

**Files:**
- modify: crates/beamr/src/distribution/mod.rs
- modify: crates/beamr/src/native/gate3_bifs/mod.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R3: erlang:is_alive/0

Returns true if the node is alive (distribution started), false otherwise.

**Acceptance:**
- is_alive() returns false in non-distributed mode (nonode@nohost)
- is_alive() returns true when distribution is configured with a real node name
- Test: verify is_alive reflects distribution state

**Files:**
- modify: crates/beamr/src/native/gate3_bifs/mod.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Boundaries

- SHALL NOT implement net_kernel as a BEAM process — implement as BIFs backed by ConnectionManager
- SHALL NOT implement automatic reconnection — manual connect_node only
- SHALL NOT implement hidden nodes or node types

# B-088: term_to_binary options and compression

> **Cluster:** beamr
> **Depends on:** B-086, B-087
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

term_to_binary/1 and binary_to_term/1 (B-086, B-087) handle the basic encode/decode cycle. Real-world use needs two extensions: (1) Compression — term_to_binary/2 with {compressed, Level} wraps the ETF payload in zlib, reducing wire/storage size significantly for large terms. The loader already uses flate2 for decompressing the LitT chunk, so the dependency exists. (2) Safe mode — binary_to_term/2 with [safe] prevents the decoder from creating new atoms. Any atom in the ETF that isn't already in the atom table causes a decode failure. This defends against atom table exhaustion from untrusted input (network messages, user-supplied binaries). Both are standard BEAM BIF options used by OTP libraries and the distribution protocol.

## Task

Add term_to_binary/2 with compression options ({compressed, Level} using flate2 zlib), add decompression support to the decoder for COMPRESSED_EXT (tag 80), and add binary_to_term/2 with safe mode (prevent new atom creation) and used mode (return {Term, BytesUsed} tuple).

## Requirements

### R1: term_to_binary/2 with compression

WHEN erlang:term_to_binary/2 is called with a list of options, THE SYSTEM SHALL parse the options: compressed (default level 6), {compressed, Level} (0-9), and {minor_version, 2} (no behavior change, compatibility). WHEN compression is requested, THE SYSTEM SHALL encode the term to ETF, compress the payload using flate2::write::ZlibEncoder, and wrap the output as version byte (131) + COMPRESSED_EXT tag (80) + uncompressed size (u32) + zlib payload. WHEN the compressed output is not smaller than the uncompressed output, THE SYSTEM SHALL return the uncompressed output. WHEN {compressed, 0} is specified, THE SYSTEM SHALL produce uncompressed output.

**Acceptance:**
- erlang:term_to_binary/2 registered as BIF
- Second argument is a list of options: compressed (default level 6), {compressed, Level} (0-9), {minor_version, 2} (compatibility, no behavior change)
- Compression uses flate2::write::ZlibEncoder (already a dependency via the loader)
- Compressed format: version byte (131) + COMPRESSED_EXT tag (80) + uncompressed size (u32) + zlib payload
- Uncompressed format: version byte (131) + direct ETF payload (same as term_to_binary/1)
- The encoder only compresses if the compressed output is smaller than the uncompressed output (no expansion)
- Test: term_to_binary(LargeTerm, [compressed]) produces smaller binary than term_to_binary(LargeTerm)
- Test: term_to_binary(X, [{compressed, 0}]) produces uncompressed output
- Test: round-trip with compression: binary_to_term(term_to_binary(X, [compressed])) recovers X

### R2: binary_to_term decompression support

WHEN decode_term encounters a COMPRESSED_EXT tag (80) after the version byte, THE SYSTEM SHALL read the uncompressed size (u32), decompress the zlib payload using flate2::read::ZlibDecoder with a bounded reader matching the decompress_bounded pattern from the loader, and decode the decompressed ETF. WHEN the claimed uncompressed size exceeds RuntimeDecodeBudget's max_heap_words * 8 bytes, THE SYSTEM SHALL return an error. WHEN the tag is not COMPRESSED_EXT, THE SYSTEM SHALL decode normally via the existing path.

**Acceptance:**
- decode_term checks for COMPRESSED_EXT tag (80) after the version byte
- If compressed: read uncompressed size (u32), decompress zlib payload using flate2::read::ZlibDecoder, then decode the decompressed ETF
- Decompression uses bounded reader (matching decompress_bounded pattern from the loader) to prevent zip bombs
- Uncompressed size limit: same as RuntimeDecodeBudget's max_heap_words * 8 bytes (reasonable bound)
- If not compressed: decode normally (existing path)
- Test: decode a compressed binary produced by R1
- Test: reject a binary claiming unreasonable uncompressed size

**Files:**
- modify: crates/beamr/src/etf/decode.rs

**Checklist:**

### R3: binary_to_term/2 with safe mode

WHEN erlang:binary_to_term/2 is called with a list of options, THE SYSTEM SHALL parse the options: safe (prevent new atom creation) and used (return {Term, UsedBytes} tuple). THE SYSTEM SHALL define a DecodeOptions struct with safe: bool and return_used: bool fields. WHEN safe mode is enabled, THE SYSTEM SHALL use atom_table.lookup(name) instead of atom_table.intern(name) in the atom decode path; if lookup returns None, THE SYSTEM SHALL return DecodeError::UnsafeAtom(name). WHEN used mode is enabled, THE SYSTEM SHALL track bytes consumed and return a 2-tuple {Term, BytesUsed} on heap.

**Acceptance:**
- erlang:binary_to_term/2 registered as BIF
- Second argument is a list of options: safe (prevent new atom creation), used (return {Term, UsedBytes} tuple)
- DecodeOptions struct with safe: bool and return_used: bool fields
- Safe mode: in the atom decode path, use atom_table.lookup(name) instead of atom_table.intern(name). If lookup returns None, return DecodeError::UnsafeAtom(name).
- Used mode: track bytes consumed, return 2-tuple {Term, BytesUsed} on heap
- Test: binary_to_term(term_to_binary(hello), [safe]) succeeds (atom 'hello' exists)
- Test: binary_to_term(Binary_with_novel_atom, [safe]) fails with badarg
- Test: binary_to_term(Bin, [used]) returns {Term, Size} where Size = byte_size(Bin)

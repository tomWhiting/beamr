# B-088: term_to_binary options and compression

> **Cluster:** beamr
> **Depends on:** B-086 (encoder), B-087 (decoder)
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

`term_to_binary/1` and `binary_to_term/1` (B-086, B-087) handle the basic encode/decode cycle. Real-world use needs two extensions:

1. **Compression** — `term_to_binary/2` with `{compressed, Level}` wraps the ETF payload in zlib, reducing wire/storage size significantly for large terms. The loader already uses `flate2` for decompressing the LitT chunk, so the dependency exists.

2. **Safe mode** — `binary_to_term/2` with `[safe]` prevents the decoder from creating new atoms. Any atom in the ETF that isn't already in the atom table causes a decode failure. This defends against atom table exhaustion from untrusted input (network messages, user-supplied binaries).

Both are standard BEAM BIF options used by OTP libraries and the distribution protocol.

## Boundaries

- This brief SHALL add `term_to_binary/2` with compression options
- This brief SHALL add `binary_to_term/2` with safe mode
- This brief SHALL NOT implement the `deterministic` option (ordering guarantees — separate concern)
- This brief SHALL NOT implement the `local` option (distribution-specific — Phase 3)

## Requirements

### R1: term_to_binary/2 with compression

Add options support to the encoder.

**Acceptance:**
- `erlang:term_to_binary/2` registered as BIF
- Second argument is a list of options:
  - `compressed` — compress at default level (6)
  - `{compressed, Level}` — compress at specified level (0–9, where 0 = no compression, 9 = max)
  - `{minor_version, 2}` — use UTF-8 atom encoding (default; included for compatibility, no behavior change since B-086 already uses UTF-8 atoms)
- Compression uses `flate2::write::ZlibEncoder` (already a dependency via the loader)
- Compressed format: version byte (131) + `COMPRESSED_EXT` tag (80) + uncompressed size (u32) + zlib payload
- Uncompressed format: version byte (131) + direct ETF payload (same as term_to_binary/1)
- The encoder only compresses if the compressed output is smaller than the uncompressed output (no expansion)
- Test: `term_to_binary(LargeTerm, [compressed])` produces smaller binary than `term_to_binary(LargeTerm)`
- Test: `term_to_binary(X, [{compressed, 0}])` produces uncompressed output
- Test: round-trip with compression: `binary_to_term(term_to_binary(X, [compressed]))` recovers X

**Files:**
- modify: crates/beamr/src/etf/encode.rs (compression wrapper)
- modify: crates/beamr/src/native/etf_bifs.rs (BIF registration and option parsing)
- modify: crates/beamr/src/etf/tags.rs (add COMPRESSED_EXT = 80)

**Checklist:** C150

### R2: binary_to_term decompression support

The decoder must handle compressed ETF input (produced by term_to_binary/2 with compression).

**Acceptance:**
- `decode_term` checks for `COMPRESSED_EXT` tag (80) after the version byte
- If compressed: read uncompressed size (u32), decompress zlib payload using `flate2::read::ZlibDecoder`, then decode the decompressed ETF
- Decompression uses bounded reader (matching `decompress_bounded` pattern from the loader) to prevent zip bombs
- Uncompressed size limit: same as RuntimeDecodeBudget's max_heap_words * 8 bytes (reasonable bound)
- If not compressed: decode normally (existing path)
- Test: decode a compressed binary produced by R1
- Test: reject a binary claiming unreasonable uncompressed size

**Files:**
- modify: crates/beamr/src/etf/decode.rs

**Checklist:** C150

### R3: binary_to_term/2 with safe mode

Add options support to the decoder, specifically the `safe` option that prevents atom creation.

**Acceptance:**
- `erlang:binary_to_term/2` registered as BIF
- Second argument is a list of options:
  - `safe` — only decode atoms that already exist in the atom table. If the ETF contains an atom not in the table, return `ExecError::Badarg`.
  - `used` — return `{Term, UsedBytes}` tuple instead of just Term, indicating how many bytes of the input were consumed. Useful for parsing multiple concatenated ETF terms.
- `DecodeOptions` struct with `safe: bool` and `return_used: bool` fields
- Safe mode: in the atom decode path, use `atom_table.lookup(name)` instead of `atom_table.intern(name)`. If lookup returns None, return `DecodeError::UnsafeAtom(name)`.
- Used mode: track bytes consumed, return 2-tuple `{Term, BytesUsed}` on heap
- Test: `binary_to_term(term_to_binary(hello), [safe])` succeeds (atom 'hello' exists)
- Test: `binary_to_term(Binary_with_novel_atom, [safe])` fails with badarg
- Test: `binary_to_term(Bin, [used])` returns `{Term, Size}` where Size = byte_size(Bin)

**Files:**
- modify: crates/beamr/src/etf/decode.rs (DecodeOptions, safe mode)
- modify: crates/beamr/src/native/etf_bifs.rs (BIF registration and option parsing)

**Checklist:** C150

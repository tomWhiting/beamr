# B-033: Gleam stdlib native stubs — string, iolist, and gleam_stdlib BIFs

> **Cluster:** beamr
> **Depends on:** B-032
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

Gleam's string operations delegate to two layers: gleam_stdlib (a NIF-like module compiled from Erlang/FFI) and Erlang's string module. The compiled gleam_stdlib.beam cannot load due to ETF tag 70 (IEEE 754 float literal), so these functions must be implemented as native Rust BIFs. Without them, any Gleam workflow using string concatenation, formatting, or result handling fails at runtime.

## Task

Implement 25 missing BIFs across 4 modules as native Rust stubs, and fix the ETF tag 70 decoder gap that blocks loading gleam_stdlib.beam and gleam@list.beam.

## Requirements

### R1: ETF tag 70 — NEW_FLOAT_EXT decoder

Fix the ETF literal decoder to handle tag 70 (IEEE 754 double, 8 bytes big-endian). This tag is used by gleam_stdlib.beam and gleam@list.beam for float constants.

**Acceptance:**
- gleam_stdlib.beam and gleam@list.beam from test-workflows/sample/ load without ETF decode errors
- Tag 70 decodes to the correct Float term value
- Existing ETF tests still pass

**Files:**
- modify: crates/beamr/src/loader/decode/chunks.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

# B-033: Gleam stdlib native stubs — string, iolist, and gleam_stdlib BIFs

> **Cluster:** beamr
> **Depends on:** B-032
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

Gleam's string operations delegate to two layers: `gleam_stdlib` (a NIF-like module compiled from Erlang/FFI) and Erlang's `string` module. The compiled `gleam_stdlib.beam` cannot load due to ETF tag 70 (IEEE 754 float literal), so these functions must be implemented as native Rust BIFs. Without them, any Gleam workflow using string concatenation, formatting, or result handling fails at runtime.

## Task

Implement 25 missing BIFs across 4 modules as native Rust stubs, and fix the ETF tag 70 decoder gap that blocks loading `gleam_stdlib.beam` and `gleam@list.beam`.

## Boundaries

- This brief SHALL NOT implement full Unicode grapheme segmentation (stub string_pop_grapheme with byte-level splitting)
- This brief SHALL NOT implement gleam_stdlib functions not imported by the sample_workflow dependency chain
- This brief SHALL fix ETF tag 70 decoding even though stubs bypass the need for loading gleam_stdlib.beam — future modules will need it

## Requirements

### R1: ETF tag 70 — NEW_FLOAT_EXT decoder

Fix the ETF literal decoder to handle tag 70 (IEEE 754 double, 8 bytes big-endian). This tag is used by `gleam_stdlib.beam` and `gleam@list.beam` for float constants.

**Acceptance:**
- `gleam_stdlib.beam` and `gleam@list.beam` from test-workflows/sample/ load without ETF decode errors
- Tag 70 decodes to the correct Float term value
- Existing ETF tests still pass

**Files:**
- modify: crates/beamr/src/loader/decode/chunks.rs (ETF decoder)

**Checklist:** C150

### R2: erlang module BIFs (2 functions)

Register under `erlang` module:
- `byte_size/1`: Return byte length of a binary term. Return badarg for non-binaries.
- `iolist_size/1`: Return byte size of an iolist (binary or nested list of binaries/integers). For the initial stub, handle binaries directly and return badarg for complex iolists.

**Acceptance:**
- `byte_size(<<"hello">>)` returns `5`
- `iolist_size(<<"hello">>)` returns `5`
- Non-binary arguments return badarg
- Tests cover each function

**Files:**
- modify: crates/beamr/src/native/gate3_bifs/mod.rs
- modify: crates/beamr/src/native/gate3_bifs/tests.rs

**Checklist:** C150

|----------|-------|-----------|
| string_replace | 3 | Replace all occurrences of pattern in string |
| less_than | 2 | Structural less-than comparison, return bool atom |

| crop_string | 2 | Drop characters before first occurrence of pattern |
| contains_string | 2 | Return true/false atom for substring containment |
| string_starts_with | 2 | Prefix test, return true/false atom |
| string_ends_with | 2 | Suffix test, return true/false atom |
| string_pop_grapheme | 1 | Split first byte from string (simplified) |
| utf_codepoint_list_to_string | 1 | Convert codepoint list to binary |
| inspect | 1 | Debug-format any term as a binary string |

| string_remove_suffix | 2 | Strip matching suffix, return {ok, Rest} or error |

**Acceptance:**

# B-027: Type conversion BIFs and process registry

> **Cluster:** beamr
> **Depends on:** B-026
> **Checklist:**
> - C141 — demonitor(Ref) removes the monitor; no DOWN message is sent after demonitor
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer
> **Stories:**
> - S4 (Workflow Author, Writing Gleam Workflows) — As a workflow author, I want to call Rust-native operations (git, AST merge, dependency graph) from Gleam via registered functions so that I get zero-overhead access to the host's capabilities.
> - S8 (Meridian Runtime, Embedding beamr for Workflow Execution) — As the Meridian runtime, I want to create a beamr VM instance, load .beam modules, and spawn processes from Rust so that I can embed workflow execution in the server process.

## Purpose

gleam_erlang_ffi imports type conversion BIFs and process registry BIFs. This brief implements them and adds a process name registry to SharedState.

## Task

Implement type conversion BIFs, process registry (register/unregister/whereis), and demonitor/2.

## Requirements

### R1: Type conversion BIFs

Implement atom_to_binary/2, binary_to_existing_atom/1, binary_to_list/1, list_to_binary/1, map_get/2.

**Acceptance:**
- atom_to_binary(hello, utf8) returns <<hello>>
- binary_to_existing_atom(<<ok>>) returns ok
- binary_to_list(<<abc>>) returns [97,98,99]
- list_to_binary([97,98,99]) returns <<abc>>
- map_get(key, #{key=>val}) returns val
- map_get(missing, #{}) returns badkey error

**Files:**
- modify: crates/beamr/src/native/gate3_bifs.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

**Stories:**
- S4 (Workflow Author, Writing Gleam Workflows) — As a workflow author, I want to call Rust-native operations (git, AST merge, dependency graph) from Gleam via registered functions so that I get zero-overhead access to the host's capabilities.

### R2: Process registry BIFs

Implement register/2, unregister/1, whereis/1 backed by HashMap<Atom, u64> on SharedState via RegistryFacility trait.

**Acceptance:**
- register(name, Pid) associates name
- whereis(name) returns Pid
- unregister(name) removes association
- whereis(unregistered) returns undefined
- register(taken, Pid) returns badarg

**Files:**
- create: crates/beamr/src/native/registry.rs
- modify: crates/beamr/src/native/gate3_bifs.rs
- modify: crates/beamr/src/scheduler/mod.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

**Stories:**
- S8 (Meridian Runtime, Embedding beamr for Workflow Execution) — As the Meridian runtime, I want to create a beamr VM instance, load .beam modules, and spawn processes from Rust so that I can embed workflow execution in the server process.

### R3: erlang:demonitor/2

demonitor/2 accepts (Ref, Options) list. Support [flush] and [info] options. Empty list behaves like demonitor/1.

**Acceptance:**
- demonitor(Ref, []) behaves like demonitor/1
- demonitor(Ref, [flush]) removes monitor and flushes DOWN
- demonitor(Ref, [info]) returns true/false
- Non-list second arg returns badarg

**Files:**
- modify: crates/beamr/src/native/gate3_bifs.rs

**Checklist:**
- C141 — demonitor(Ref) removes the monitor; no DOWN message is sent after demonitor

**Stories:**
- S8 (Meridian Runtime, Embedding beamr for Workflow Execution) — As the Meridian runtime, I want to create a beamr VM instance, load .beam modules, and spawn processes from Rust so that I can embed workflow execution in the server process.

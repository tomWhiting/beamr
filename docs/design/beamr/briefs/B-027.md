# B-027: Type conversion BIFs and process registry

> **Cluster:** beamr
> **Depends on:** B-026
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer
> **Stories:**
> - S4 (Workflow Author) — As a workflow author, I want to call Rust-native operations from Gleam via registered functions.
> - S8 (Meridian Runtime) — As the Meridian runtime, I want to create a beamr VM instance, load .beam modules, and spawn processes from Rust.

## Purpose

gleam_erlang_ffi imports type conversion BIFs (atom_to_binary, binary_to_existing_atom, binary_to_list, list_to_binary, map_get) and process registry BIFs (register/2, unregister/1, whereis/1). It also uses demonitor/2 (2-arg variant with options, vs our existing /1). These are needed before gleam_erlang_ffi can run.

## Task

Implement type conversion BIFs and a process name registry. The registry is a `HashMap<Atom, u64>` on SharedState, protected by a Mutex. Type conversion BIFs use the AtomTable for atom<->binary conversions. Register all via `register_gate3_bifs()` alongside B-026's BIFs.

## Boundaries

- This brief SHALL NOT implement register/2 for ports (only processes)
- This brief SHALL NOT implement binary_to_term/1 or term_to_binary/1 (ETF serialization)
- This brief SHALL NOT implement binary_to_integer or binary_to_float (number parsing)

## Requirements

### R1: Type conversion BIFs

Implement the following erlang BIFs:
- `atom_to_binary/2`: Convert atom to UTF-8 binary. Second arg is encoding (accept `utf8` or `latin1`, both produce the same result since atom names are ASCII).
- `binary_to_existing_atom/1`: Look up binary string in the atom table. Return the atom if it exists, `badarg` if not interned.
- `binary_to_list/1`: Convert binary to a list of integer code points.
- `list_to_binary/1`: Convert a list of integers (0-255) to a binary.
- `map_get/2`: Extract a value from a map by key. Return `badkey` error if key not found.

**Acceptance:**
- atom_to_binary(hello, utf8) returns <<"hello">>
- binary_to_existing_atom(<<"ok">>) returns ok (if interned)
- binary_to_existing_atom(<<"nonexistent">>) returns badarg
- binary_to_list(<<"abc">>) returns [97, 98, 99]
- list_to_binary([97, 98, 99]) returns <<"abc">>
- map_get(key, #{key => val}) returns val
- map_get(missing, #{}) returns {badkey, missing} error

**Files:**
- modify: crates/beamr/src/native/gate3_bifs.rs

**Checklist:**
- C150

### R2: Process registry BIFs

Implement:
- `register/2`: Associate an atom name with a PID. Fail if name already registered or PID already has a name.
- `unregister/1`: Remove a name registration. Fail if name not registered.
- `whereis/1`: Look up a PID by registered name. Return the PID or `undefined`.

The registry SHALL be a `HashMap<Atom, u64>` on the scheduler's SharedState, accessed via a new `RegistryFacility` trait on ProcessContext.

**Acceptance:**
- register(name, Pid) associates name with Pid
- whereis(name) returns the Pid
- unregister(name) removes the association
- whereis(unregistered) returns undefined
- register(already_taken, Pid) returns badarg
- Registered names survive across scheduler ticks

**Files:**
- modify: crates/beamr/src/native/gate3_bifs.rs
- modify: crates/beamr/src/scheduler/mod.rs (add registry to SharedState)
- create: crates/beamr/src/native/registry.rs (RegistryFacility trait)

**Checklist:**
- C150

### R3: erlang:demonitor/2

`erlang:demonitor/2` SHALL accept (Ref, Options) where Options is a list. For now, accept `[flush]` and `[info]` options. The `flush` option flushes any pending DOWN message from the caller's mailbox. The `info` option returns `true` if the monitor was found, `false` if not (instead of silently succeeding). Empty options list `[]` behaves like `demonitor/1`.

**Acceptance:**
- demonitor(Ref, []) behaves like demonitor/1
- demonitor(Ref, [flush]) removes monitor and flushes DOWN
- demonitor(Ref, [info]) returns true if found, false if not
- Non-list second argument returns badarg

**Files:**
- modify: crates/beamr/src/native/gate3_bifs.rs

**Checklist:**
- C141

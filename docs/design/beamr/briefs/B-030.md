# B-030: gleam_erlang_ffi — selector system

> **Cluster:** beamr
> **Depends on:** B-024, B-028b
> **Checklist:**
> - C83 — loop_rec, loop_rec_end, remove_message instructions: selective receive mailbox scan
> - C127 — Selective receive scans the mailbox from the save pointer forward, testing each message against patterns
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer
> **Stories:**
> - S5 (Workflow Author, Writing Gleam Workflows) — As a workflow author, I want workflow steps to communicate via typed message passing so that data flows between steps without shared mutable state or serialization.
> - S7 (Workflow Author, Writing Gleam Workflows) — As a workflow author, I want to set timeouts on receive expressions so that a step waiting for input does not hang indefinitely if its dependency fails silently.

## Purpose

gleam_erlang's process module wraps selective receive with a typed selector abstraction. This is the foundation of gleam_otp's actor message loop.

## Task

Implement selector system as native Rust BIFs under gleam_erlang_ffi MFAs.

## Requirements

### R1: new_selector/0 and basic structure

new_selector/0 returns empty list. Selector is a list of {Tag, Handler} tuples.

**Acceptance:**
- new_selector() returns NIL
- Selector is a standard BEAM list

**Files:**
- create: crates/beamr/src/native/selector_ffi.rs
- modify: crates/beamr/src/native/mod.rs

**Stories:**

### R2: insert/map/merge/remove selector handlers

insert_selector_handler/3 prepends entry, map_selector/2 wraps handlers, merge_selector/2 concatenates, remove_selector_handler/2 removes by tag.

**Acceptance:**
- insert adds handler
- map transforms outputs
- merge combines selectors
- remove removes by tag
- Operations compose correctly

**Files:**
- modify: crates/beamr/src/native/selector_ffi.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

**Stories:**
- S5 (Workflow Author, Writing Gleam Workflows) — As a workflow author, I want workflow steps to communicate via typed message passing so that data flows between steps without shared mutable state or serialization.

### R3: select/1 and select/2

select/1 scans mailbox against selector handlers. First match consumes message and returns handler result. No match suspends. select/2 adds timeout.

**Acceptance:**
- Matching message returns handler result
- No match suspends process
- Timeout returns {error, nil}
- Non-matching messages stay in mailbox
- First matching handler wins

# B-097: ets:match and ets:select — pattern-based queries

> **Cluster:** beamr
> **Depends on:** B-096
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

`ets:match` and `ets:select` are the pattern-based query BIFs for ETS tables. `match/2` uses a simplified pattern (no guards, returns bound variables). `select/2` uses full match specifications (pattern + guards + body). Both support continuation-based pagination via `/3` variants.

## Task

Implement match/2, match/3, select/2, select/3, match_object/2, and match_delete/2 BIFs.

## Requirements

### R1: ets:match/2 and ets:match_object/2

ets:match/2 and ets:match_object/2

**Acceptance:**
- `ets:match(Tab, Pattern)` scans table, returns list of bound variable lists for matching entries
- `ets:match_object(Tab, Pattern)` returns the full matching tuples (not just bound variables)
- Pattern uses `'_'` for don't-care and `'$N'` for binding
- Test: table with {a,1},{b,2},{c,3}, match({_, '$1'}) → [[1],[2],[3]]

**Files:**
- modify: crates/beamr/src/native/ets_bifs.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R2: ets:select/2

ets:select/2

**Acceptance:**
- `ets:select(Tab, MatchSpec)` runs a compiled match spec against all table entries
- Returns list of results from the match spec body for all matching entries
- Test: select with guard `[{{'$1','$2'},[{'>','$1',1}],['$2']}]` on table {1,a},{2,b},{3,c} → [b, c]

**Files:**
- modify: crates/beamr/src/native/ets_bifs.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R3: Continuation-based pagination

Continuation-based pagination

**Acceptance:**
- `ets:match(Tab, Pattern, Limit)` returns `{Results, Continuation}` or `'$end_of_table'`
- `ets:match(Continuation)` resumes from where the previous call left off
- Same for `ets:select/3` and `ets:select/1`
- Continuations are opaque terms (tuples encoding table ID, position, pattern/spec)
- Test: table with 100 entries, match with limit 10 → 10 results + continuation, call 10 times → all 100

**Files:**
- modify: crates/beamr/src/native/ets_bifs.rs
- modify: crates/beamr/src/ets/match_spec.rs (continuation state)

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R4: ets:match_delete/2

ets:match_delete/2

**Acceptance:**
- `ets:match_delete(Tab, Pattern)` deletes all entries matching the pattern, returns true
- Uses the same pattern matching as ets:match
- Test: match_delete({_, 1}) removes all entries with 1 as second element

**Files:**
- modify: crates/beamr/src/native/ets_bifs.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

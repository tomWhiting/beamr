# B-073: Binary size threshold and automatic promotion to refc

> **Cluster:** beamr
> **Depends on:** B-072 (SharedBinary and ProcBin)
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

With ProcBin available (B-072), the system needs a decision point: when should a binary be stored inline vs. off-heap? BEAM uses 64 bytes as the threshold — binaries <= 64 bytes are heap binaries, > 64 bytes become refc binaries. This brief implements that threshold at every binary creation point: `write_binary()`, BinaryBuilder finalization, and BIF results that produce binaries.

## Task

Add a configurable size threshold. Modify all binary creation paths to check the threshold and route to either inline `write_binary()` or `write_proc_bin()`.

## Boundaries

- SHALL NOT change the threshold at runtime — it is set at VM creation and fixed
- SHALL NOT modify the inline Binary format — small binaries are unchanged
- SHALL NOT implement sub-binary promotion — that's B-074
- The default threshold SHALL be 64 bytes (matching BEAM)

## Requirements

### R1: Configurable threshold constant

Define the binary size threshold as a configurable constant.

**Acceptance:**
- `const REFC_BINARY_THRESHOLD: usize = 64` defined in term/binary.rs or term/shared_binary.rs
- Binaries with `len <= REFC_BINARY_THRESHOLD` are stored inline
- Binaries with `len > REFC_BINARY_THRESHOLD` are stored as ProcBin
- The threshold is documented

**Files:**
- modify: crates/beamr/src/term/shared_binary.rs

**Checklist:** C150

### R2: Smart binary allocation function

Create a single entry point that decides inline vs. refc based on the threshold.

**Acceptance:**
- `alloc_binary(heap, bytes) -> Option<Term>` checks byte length against threshold
- If `<= threshold`: calls existing `write_binary(heap, bytes)` (inline)
- If `> threshold`: creates SharedBinary, calls `write_proc_bin(heap, shared)` (refc)
- Returns the same `Term` type regardless of storage strategy
- All callers get transparent inline/refc selection
- Test: 64-byte binary → inline (Binary tag 0x16)
- Test: 65-byte binary → refc (ProcBin tag 0x19)
- Test: 0-byte binary → inline

**Files:**
- modify: crates/beamr/src/term/shared_binary.rs

**Checklist:** C150

### R3: BinaryBuilder finalization uses threshold

Modify `finalize_builder()` to route through `alloc_binary()` so large constructed binaries automatically become refc.

**Acceptance:**
- BinaryBuilder producing <= 64 bytes finalizes to inline Binary
- BinaryBuilder producing > 64 bytes finalizes to ProcBin
- Existing bs_init/bs_put/finalize pipeline works unchanged
- Test: build a 100-byte binary via bs_put instructions → result is ProcBin

**Files:**
- modify: crates/beamr/src/interpreter/opcodes/binary.rs

**Checklist:** C150

### R4: BIF binary results use threshold

Modify BIF helpers that produce binary results (make_leaked_binary, iodata_append, string operations) to use the threshold.

**Acceptance:**
- BIF results that produce binaries > 64 bytes automatically become ProcBin
- Small BIF results remain inline
- Existing BIF tests continue to pass
- Test: `iodata_append` producing > 64 bytes returns ProcBin

**Files:**
- modify: crates/beamr/src/native/stdlib_stubs/mod.rs (make_leaked_binary)

**Checklist:** C150

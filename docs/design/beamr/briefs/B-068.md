# B-068: Binary ops expansion — bs_skip, bs_get_float, UTF ops, bs_match

> **Cluster:** beamr
> **Depends on:** B-023a, B-023b, B-023c
> **Checklist:**
> - C73 — get_hd and get_tl instructions: extract head or tail from a cons cell

## Purpose

The binary matching infrastructure (B-023a/b/c) landed with 8 of 21 BinaryOp variants implemented across 6 match arms: BsInitWritable, BsCreateBin (construction), BsStartMatch3, BsStartMatch4 (match context setup), BsGetInteger2 (integer extraction), BsGetBinary2 (sub-binary extraction), BsMatchString (literal matching), and BsTestTail2 (end-of-binary check). The remaining 13 variants fall through to UnsupportedOpcode. These block any BEAM code that uses float binary patterns, UTF string matching, bit skipping, or the OTP 25+ compound bs_match instruction. Gleam's bit array patterns compile to several of these opcodes. binary.rs is currently 621 lines. Adding 13 handlers would push it far over the 500-line limit. This brief splits the binary operations into focused submodules.

## Task

Split binary.rs into focused submodules (mod.rs, construction.rs, matching.rs, utf.rs), then implement the 13 missing BinaryOp handlers: BsSkipBits2, BsTestUnit, BsGetFloat2, BsGetTail, BsGetUtf8, BsSkipUtf8, BsGetUtf16, BsSkipUtf16, BsGetUtf32, BsSkipUtf32, BsGetPosition, BsSetPosition, and the compound BsMatch instruction.

### R2: Skip and test operations — BsSkipBits2, BsTestUnit

WHEN the interpreter encounters BsSkipBits2, THE SYSTEM SHALL advance the match context position by Size * Unit bits without reading, branching to Fail if insufficient bits remain. WHEN the interpreter encounters BsTestUnit, THE SYSTEM SHALL test that the remaining bits in the match context are divisible by Unit, branching to Fail if not.

**Acceptance:**

- BsTestUnit: test that the remaining bits in the match context are divisible by Unit. Operands: [Fail, MatchContext, Unit]. Branch to Fail if not.

- Test: skip 16 bits in a 32-bit binary, verify position advanced
- Test: skip past end of binary, verify branch to fail

### R3: Float extraction and tail operations — BsGetFloat2, BsGetTail

WHEN the interpreter encounters BsGetFloat2, THE SYSTEM SHALL extract an IEEE 754 float from the match context where Size * Unit must be 32 or 64 bits, creating a boxed float on the process heap. WHEN the interpreter encounters BsGetTail, THE SYSTEM SHALL extract the remaining bytes of the match context as a binary, creating a sub-binary or copy on the heap.

**Acceptance:**

- BsGetTail: extract the remaining bytes of the match context as a binary. Operands: [Fail, MatchContext, Live, Destination]. Creates a sub-binary or copy on the heap.

### R4: UTF encoding operations — get and skip for UTF-8/16/32

WHEN the interpreter encounters a UTF get operation (BsGetUtf8, BsGetUtf16, BsGetUtf32), THE SYSTEM SHALL read one encoded codepoint from the match context and store it as an integer, branching to Fail if the encoding is invalid. WHEN the interpreter encounters a UTF skip operation (BsSkipUtf8, BsSkipUtf16, BsSkipUtf32), THE SYSTEM SHALL advance past one encoded codepoint without storing, branching to Fail if invalid. THE SYSTEM SHALL validate all codepoints by rejecting surrogates (0xD800..0xDFFF) and values greater than 0x10FFFF.

**Acceptance:**
- BsGetUtf8: read one UTF-8 encoded codepoint from the match context, store as integer. Operands: [Fail, MatchContext, Live, Flags, Destination]. Branch to Fail if invalid UTF-8 at current position.
- BsSkipUtf8: skip one UTF-8 encoded codepoint without storing. Branch to Fail if invalid.
- BsGetUtf16: read one UTF-16 encoded codepoint (2 or 4 bytes depending on surrogate pairs). Flags include endianness (big/little/native).
- BsSkipUtf16: skip one UTF-16 codepoint.
- BsGetUtf32: read one UTF-32 codepoint (always 4 bytes). Validate codepoint range (0..0x10FFFF, excluding surrogates).
- BsSkipUtf32: skip one UTF-32 codepoint with validation.
- All UTF ops must validate codepoints: reject surrogates (0xD800..0xDFFF), values > 0x10FFFF
- Test: UTF-8 decode of ASCII (1 byte), 2-byte, 3-byte, and 4-byte sequences
- Test: UTF-8 invalid sequence branches to fail
- Test: UTF-16 big-endian and little-endian decode
- Test: UTF-32 with valid and invalid codepoints

### R5: Match context navigation — BsGetPosition, BsSetPosition

WHEN the interpreter encounters BsGetPosition, THE SYSTEM SHALL read the match context's current bit offset as an integer term and store it in the destination register. WHEN the interpreter encounters BsSetPosition, THE SYSTEM SHALL set the match context's current bit offset from an integer term in the source register.

**Acceptance:**
- BsGetPosition: read the match context's current bit offset as an integer term and store in the destination register. Operands: [MatchContext, Destination, Live].
- BsSetPosition: set the match context's current bit offset from an integer term in the source register. Operands: [MatchContext, Source].
- These support the compiler's backtracking optimization for complex binary patterns

### R6: Compound binary match — BsMatch

WHEN the interpreter encounters BsMatch (opcode 182), THE SYSTEM SHALL process a fail label, a match context, and a list of match commands encoded as operands, executing each sub-command sequentially. WHEN any sub-command fails, THE SYSTEM SHALL branch to Fail with the match context position unchanged (entire match is atomic).

**Acceptance:**
- BsMatch: takes a fail label, a match context, and a list of match commands encoded as operands. Operands: [Fail, MatchContext, Commands...] where Commands is a nested operand list.
- Each command type maps to an equivalent individual operation: {ensure, Live, Bits} — check remaining bits; {integer, Live, Flags, Size, Unit, Dst} — like BsGetInteger2; {float, Live, Flags, Size, Unit, Dst} — like BsGetFloat2; {binary, Live, Flags, Size, Unit, Dst} — like BsGetBinary2; {skip, Stride} — like BsSkipBits2; {get_tail, Live, Dst} — like BsGetTail; {'=:=', Live, Bits, Value} — match exact value
- If any sub-command fails, branch to Fail (entire match is atomic — match context position is not partially advanced on failure)
- The operand encoding follows the BEAM extended operand format — reference OTP source lib/compiler/src/beam_disasm.erl and erts/emulator/beam/jit/beam_jit_common.cpp (bs_match handler) for the encoding spec rather than reverse-engineering from compiled .beam files
- Test: compound match extracting integer + skip + binary in sequence
- Test: compound match with failing sub-command branches to fail, position unchanged

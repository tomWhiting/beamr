# B-026: Gate 3 erlang BIFs — element, send, make_ref, spawn/1, type queries

> **Cluster:** beamr
> **Depends on:** B-023a, B-024
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer
> - C82 — send instruction: send the term in x(1) to the process identified by x(0)
> **Stories:**
> - S4 (Workflow Author) — As a workflow author, I want to call Rust-native operations from Gleam via registered functions.
> - S8 (Meridian Runtime) — As the Meridian runtime, I want to create a beamr VM instance, load .beam modules, and spawn processes from Rust.

## Purpose

gleam_erlang and gleam_otp import several erlang BIFs beyond what Gate 1+2 provide. These are all straightforward: `element/2` is tuple indexing, `send/2` is the BIF form of `!`, `make_ref/0` is a unique reference counter, `spawn/1` and `spawn_link/1` take a fun instead of MFA, `is_process_alive/1` checks the process table, and `tuple_size/1` returns the arity. This brief registers them as Gate 3 BIFs.

## Task

Implement 7 BIFs in a new `crates/beamr/src/native/gate3_bifs.rs` file and register them via `register_gate3_bifs()`. These BIFs use the existing NativeFn signature and ProcessContext facilities.

## Boundaries

- This brief SHALL NOT implement spawn_monitor/1 or spawn_opt/2
- This brief SHALL NOT implement send/3 (with options)
- This brief SHALL NOT implement element/3 (setelement)

## Requirements

### R1: erlang:element/2

`erlang:element/2` SHALL accept (Index, Tuple) where Index is a 1-based small integer and Tuple is a tuple term. It SHALL return the element at the given index. Out-of-range index or non-tuple second argument SHALL return `Err(badarg)`.

**Acceptance:**
- element(1, {a, b, c}) returns a
- element(3, {a, b, c}) returns c
- element(0, {a}) returns badarg (1-based)
- element(1, not_a_tuple) returns badarg

**Files:**
- create: crates/beamr/src/native/gate3_bifs.rs
- modify: crates/beamr/src/native/mod.rs

**Checklist:**
- C150

### R2: erlang:send/2 and erlang:tuple_size/1

`erlang:send/2` SHALL accept (Pid, Message) and send Message to the target process's mailbox, returning Message. This is the BIF form of the `!` operator. `erlang:tuple_size/1` SHALL accept a tuple and return its arity as a small integer.

**Acceptance:**
- send(Pid, msg) delivers msg to target mailbox and returns msg
- send to nonexistent pid is a no-op returning msg (BEAM semantics)
- tuple_size({a, b, c}) returns 3
- tuple_size(not_a_tuple) returns badarg

**Files:**
- create: crates/beamr/src/native/gate3_bifs.rs

**Checklist:**
- C82
- C150

### R3: erlang:make_ref/0, erlang:is_process_alive/1, erlang:spawn/1, erlang:spawn_link/1

`erlang:make_ref/0` SHALL return a unique reference (small integer from a monotonic counter on ProcessContext). `erlang:is_process_alive/1` SHALL accept a PID and return `true` if the process exists and is not exited, `false` otherwise. `erlang:spawn/1` and `erlang:spawn_link/1` SHALL accept a zero-arity fun and spawn a process that calls it. For now, the fun MUST be a code pointer (MFA export); arbitrary closures are not yet supported — return `badarg` for non-export funs with a clear error.

**Acceptance:**
- make_ref() returns unique values on successive calls
- is_process_alive(living_pid) returns true
- is_process_alive(dead_pid) returns false
- spawn(fun module:function/0) spawns and returns pid
- spawn_link(fun module:function/0) spawns with link
- spawn(closure_with_captures) returns badarg (documented limitation)
- All 7 BIFs registered under erlang module via register_gate3_bifs()

**Files:**
- create: crates/beamr/src/native/gate3_bifs.rs
- modify: crates/beamr/src/native/mod.rs
- modify: crates/beamr-cli/src/main.rs (call register_gate3_bifs)

**Checklist:**
- C150

# B-122: Remote links — LINK/UNLINK/EXIT across nodes

> **Cluster:** beamr
> **Depends on:** B-119
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

Cross-node links use LINK and UNLINK control messages for establishment/removal, and EXIT for signal propagation. When a linked process on node B exits, the EXIT control message propagates the signal to node A. Node disconnection is treated as exit reason 'noconnection' for all linked processes on the disconnected node.

## Task

Implement cross-node link lifecycle with exit signal propagation.

## Requirements

### R1: Cross-node link establishment and removal

Route link/unlink BIFs through distribution when the target is a remote PID.

**Acceptance:**
- link(RemotePid) sends LINK control to remote node
- Remote node establishes the link locally
- unlink(RemotePid) sends UNLINK control
- Test: link processes on different nodes, verify bidirectional

**Files:**
- modify: crates/beamr/src/distribution/control.rs
- modify: crates/beamr/src/native/process_bifs/mod.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R2: Cross-node exit signal propagation

Exit signals for linked processes propagate via EXIT control messages.

**Acceptance:**
- Process exit with remote links sends EXIT control to each remote node
- Remote node delivers the exit signal to linked processes
- Trap_exit processes receive {EXIT, RemotePid, Reason} messages
- Test: linked processes on A and B, A crashes, B receives exit signal

**Files:**
- modify: crates/beamr/src/distribution/control.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R3: Node disconnection as noconnection exit

When a distribution connection drops, treat it as exit reason 'noconnection' for all links to processes on the disconnected node.

**Acceptance:**
- Connection failure triggers 'noconnection' exit signal for all locally-linked processes that have links to the disconnected node
- Monitors receive DOWN with reason 'noconnection'
- Test: disconnect node B, all processes on A linked to B receive noconnection exit

**Files:**
- modify: crates/beamr/src/distribution/connection.rs
- modify: crates/beamr/src/distribution/control.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Boundaries

- SHALL NOT implement unlink acknowledgment protocol (UNLINK_ID, OTP 26+) — use basic UNLINK
- SHALL treat node disconnection as atomic — all links/monitors to that node fail simultaneously

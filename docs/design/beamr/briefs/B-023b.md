# B-023b: Link management BIFs — link, unlink, process_flag

> **Cluster:** beamr
> **Depends on:** B-023a
> **Checklist:**
> - C133 — link(PidA, PidB) creates a bidirectional link: both processes' link sets contain the other
> - C137 — A process with trap_exit=true receives exit signals as {'EXIT', Pid, Reason} messages in its mailbox instead of dying
> - C138 — An exit signal with reason 'kill' terminates the target process unconditionally, even if it traps exits
> - C142 — unlink(PidA, PidB) removes the bidirectional link; no exit signal is sent after unlink
> - C144 — Linking to a process that has already exited immediately sends an exit signal
> **Stories:**
> - S6 (Workflow Author, Writing Gleam Workflows) — As a workflow author, I want to define supervision trees over my workflow steps so that failed steps are automatically restarted according to a declared strategy.

## Purpose

Link and trap_exit data structures exist but are not callable from bytecode. This brief adds link/1, unlink/1, and process_flag/2 as BIFs.

## Task

Add three BIFs to process_bifs.rs. Extend ProcessContext with process table and link state access. Register in register_gate2_bifs().

## Requirements

### R1: erlang:link/1

erlang:link/1 SHALL establish a bidirectional link. Nonexistent target sends noproc. Already-linked or self-link is a no-op.

**Acceptance:**
- link(Pid) adds to both link sets
- link to nonexistent sends noproc exit signal
- link to already-linked is no-op
- link to self is no-op

**Files:**
- modify: crates/beamr/src/native/process_bifs.rs

**Checklist:**
- C133 — link(PidA, PidB) creates a bidirectional link: both processes' link sets contain the other
- C144 — Linking to a process that has already exited immediately sends an exit signal

**Stories:**
- S6 (Workflow Author, Writing Gleam Workflows) — As a workflow author, I want to define supervision trees over my workflow steps so that failed steps are automatically restarted according to a declared strategy.

### R2: erlang:unlink/1

erlang:unlink/1 SHALL remove the bidirectional link. No-op if no link exists.

**Acceptance:**
- unlink removes both directions
- unlink of nonexistent is no-op returning true
- After unlink, target exit does not affect caller

**Files:**
- modify: crates/beamr/src/native/process_bifs.rs

**Checklist:**
- C142 — unlink(PidA, PidB) removes the bidirectional link; no exit signal is sent after unlink

**Stories:**
- S6 (Workflow Author, Writing Gleam Workflows) — As a workflow author, I want to define supervision trees over my workflow steps so that failed steps are automatically restarted according to a declared strategy.

### R3: erlang:process_flag/2

erlang:process_flag/2 SHALL accept (trap_exit, Bool), set flag, return previous value. kill bypasses trap_exit.

**Acceptance:**
- process_flag(trap_exit, true) enables trapping, returns previous value
- process_flag(trap_exit, false) disables trapping
- With trap_exit, linked death delivers {EXIT, Pid, Reason}
- kill terminates regardless
- Unrecognized flag returns badarg

**Files:**
- modify: crates/beamr/src/native/process_bifs.rs

**Checklist:**

## Boundaries

- This brief SHALL NOT implement priority inheritance or priority changes via process_flag
- This brief SHALL NOT implement process groups (erlang:group_leader/2)
- This brief SHALL NOT implement erlang:process_info/1 or erlang:process_info/2

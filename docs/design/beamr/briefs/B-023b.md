# B-023b: Link management BIFs — link, unlink, process_flag

> **Cluster:** beamr
> **Depends on:** B-023a
> **Checklist:**
> - C133 — link(PidA, PidB) creates a bidirectional link
> - C137 — A process with trap_exit=true receives exit signals as {'EXIT', Pid, Reason} messages
> - C138 — An exit signal with reason 'kill' terminates unconditionally, even if it traps exits
> - C142 — unlink(PidA, PidB) removes the bidirectional link
> - C144 — Linking to a process that has already exited immediately sends an exit signal
> **Stories:**
> - S6 (Workflow Author) — As a workflow author, I want to define supervision trees over my workflow steps so that failed steps are automatically restarted according to a declared strategy.

## Purpose

The data structures for bidirectional links and the `trap_exit` flag exist on the `Process` struct, but running BEAM code cannot call them. This brief adds `link/1`, `unlink/1`, and `process_flag/2` as BIFs, extending the `ProcessContext` with supervision state access established in B-023a. These BIFs enable Gleam code to build the link graph and set trap_exit — prerequisites for supervision trees.

## Task

Add three BIFs to `crates/beamr/src/native/process_bifs.rs` (created in B-023a). Extend `ProcessContext` with access to the process table and link state so that link/unlink can modify both the caller's and target's link sets. Register all three in `register_gate2_bifs()`.

## Boundaries

- This brief SHALL NOT implement priority inheritance or priority changes via process_flag
- This brief SHALL NOT implement process groups (erlang:group_leader/2)
- This brief SHALL NOT implement erlang:process_info/1 or erlang:process_info/2

## Requirements

### R1: erlang:link/1

`erlang:link/1` SHALL accept a PID and establish a bidirectional link between the calling process and the target. If the target PID does not exist, the calling process SHALL receive an exit signal with reason `noproc` (or be terminated if it does not trap exits). If a link already exists, the call SHALL be a no-op returning `true`. Link to self SHALL be a no-op.

**Acceptance:**
- link(Pid) adds Pid to caller link set and caller PID to target link set
- link to nonexistent process sends noproc exit signal
- link to already-linked process is a no-op
- link to self is a no-op

**Files:**
- modify: crates/beamr/src/native/process_bifs.rs

**Checklist:**
- C133
- C144

### R2: erlang:unlink/1

`erlang:unlink/1` SHALL accept a PID and remove the bidirectional link between the calling process and the target. If no link exists, the call SHALL be a no-op returning `true`. After unlink, exit signals SHALL NOT propagate between the two processes.

**Acceptance:**
- unlink(Pid) removes both directions of the link
- unlink of nonexistent process is a no-op returning true
- After unlink, target exit does not affect caller

**Files:**
- modify: crates/beamr/src/native/process_bifs.rs

**Checklist:**
- C142

### R3: erlang:process_flag/2

`erlang:process_flag/2` SHALL accept `(trap_exit, Bool)` where Bool is `true` or `false`. It SHALL set the calling process's `trap_exit` flag and return the previous value as a boolean atom. With `trap_exit` set to `true`, exit signals from linked processes SHALL be delivered as `{EXIT, Pid, Reason}` messages instead of terminating the process. The `kill` signal SHALL bypass `trap_exit`.

**Acceptance:**
- process_flag(trap_exit, true) enables trapping, returns previous value
- process_flag(trap_exit, false) disables trapping
- With trap_exit=true, linked process death delivers {EXIT, Pid, Reason} message
- kill terminates regardless of trap_exit
- Unrecognized flag name returns badarg

**Files:**
- modify: crates/beamr/src/native/process_bifs.rs

**Checklist:**
- C137
- C138

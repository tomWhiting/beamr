# B-096: Match specification compiler

> **Cluster:** beamr
> **Depends on:** B-094
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

Match specifications are the query language of ETS. A match spec is a list of `{MatchHead, Guards, Body}` tuples that describe patterns to match, guard conditions to filter, and result transformations. The compiler converts this Erlang term representation into an executable form. OTP uses match specs extensively in `ets:select`, `ets:match`, trace patterns, and `dbg`.

## Task

Implement a match specification compiler that takes the term representation and produces an executable `CompiledMatchSpec` that can be evaluated against table entries.

## Requirements

### R1: Match spec parsing

Parse the term representation `[{MatchHead, Guards, Body}]` into a structured `MatchSpec`.

**Acceptance:**
- `MatchHead` is a tuple pattern with variables ($1, $2, ... as atoms or special terms) and literals
- `_` matches anything (don't-care)
- `'$_'` in body returns the entire matched object
- `'$$'` in body returns all bound variables as a list
- Guards are nested lists of guard function calls: `[{'>',  '$1', 10}]`
- Test: parse `[{{'$1','$2'}, [{'>', '$1', 10}], ['$2']}]` — matches 2-tuples where element 1 > 10, returns element 2

**Files:**
- create: crates/beamr/src/ets/match_spec.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R2: Match spec evaluation

Evaluate a compiled match spec against a tuple, returning the result or no-match.

**Acceptance:**
- `CompiledMatchSpec::eval(tuple: Term) -> Option<Term>` tests the pattern, checks guards, produces result
- Variables bind to corresponding tuple positions
- Guards are evaluated with bound variables
- Body constructs the result term from bound variables and literals
- Test: spec `[{{'$1','$2'}, [{'>', '$1', 0}], [{{'$2', '$1'}}]}]` against `{5, hello}` → returns `{hello, 5}`
- Test: same spec against `{-1, hello}` → no match (guard fails)

**Files:**
- modify: crates/beamr/src/ets/match_spec.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R3: ets:match_spec_compile/1 and ets:match_spec_run/2

Register BIFs for explicit match spec compilation and execution (used by OTP tracing).

**Acceptance:**
- `ets:match_spec_compile(Spec)` compiles a match spec term, returns a compiled reference
- `ets:match_spec_run(List, CompiledSpec)` runs the spec against each element, returns matching results
- Test: compile + run against a list of tuples produces correct matches

**Files:**

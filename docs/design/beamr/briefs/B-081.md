# B-081: system_info/1 BIF — VM introspection

> **Cluster:** beamr
> **Depends on:** B-032 (scheduler)
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

`erlang:system_info/1` returns information about the running VM — scheduler count, process count, word size, OTP version, etc. OTP's `init`, `code`, and `application` modules call it during startup. Without it, loading OTP modules that query VM properties fails.

## Task

Implement system_info/1 as a BIF with the subset of items needed by OTP libraries.

## Boundaries

- SHALL NOT implement all ~60 system_info items — only the subset needed by OTP stdlib
- SHALL return beamr-specific values where appropriate (e.g., otp_release returns a beamr version string, not an Erlang version)
- SHALL NOT implement runtime-modifiable system flags — read-only queries only

## Requirements

### R1: system_info/1 BIF with core items

Register `erlang:system_info/1`. Returns a value based on the item atom.

**Acceptance:**
- `system_info(schedulers)` → number of normal scheduler threads
- `system_info(process_count)` → number of alive processes
- `system_info(process_limit)` → maximum process count (configurable, default 262144)
- `system_info(wordsize)` → 8 (64-bit)
- `system_info(otp_release)` → binary string "27" (compatibility claim)
- `system_info(version)` → binary string with beamr version
- `system_info(system_architecture)` → binary string (e.g., "x86_64-apple-darwin")
- `system_info(atom_count)` → current atom table size
- `system_info(atom_limit)` → maximum atom count
- Unknown item returns badarg
- Test: each supported item returns the expected type

**Files:**
- create: crates/beamr/src/native/system_info_bifs.rs
- modify: crates/beamr/src/native/mod.rs

**Checklist:** C150

### R2: Scheduler and process count access

system_info needs access to scheduler and process table metrics. Add accessors to SharedState.

**Acceptance:**
- `SharedState::process_count() -> usize` returns alive process count
- `SharedState::scheduler_count() -> usize` returns normal scheduler thread count
- `SharedState::atom_count() -> usize` returns current atom table size
- These are read-only, lock-free where possible (use existing DashMap::len() etc.)
- Test: verify counts match expected values

**Files:**
- modify: crates/beamr/src/scheduler/mod.rs

**Checklist:** C150

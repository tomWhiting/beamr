# B-028: Stdlib stubs — lists, maps, timer as native BIFs

> **Cluster:** beamr
> **Depends on:** B-026
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer
> **Stories:**
> - S4 (Workflow Author) — As a workflow author, I want to call Rust-native operations from Gleam via registered functions.
> - S9 (Meridian Runtime) — As the Meridian runtime, I want to register Rust functions as callable BIFs and NIFs.

## Purpose

gleam_erlang_ffi and gleam_otp import OTP standard library functions: `lists:map/2`, `lists:foldr/3`, `maps:from_list/1`, `maps:map/2`, `maps:merge/2`, `maps:remove/2`, `timer:sleep/1`. Rather than loading the OTP .beam files (which would recursively pull in the entire Erlang stdlib), these are implemented as native Rust functions registered under the correct module:function/arity paths. Each stub must have correct semantics — not just type-correct signatures.

## Task

Implement stdlib stubs in `crates/beamr/src/native/stdlib_stubs.rs`. Register them under their OTP module names (lists, maps, timer) in the BIF registry. These are NOT erlang module BIFs — they are registered under `lists`, `maps`, and `timer` atoms.

## Boundaries

- This brief SHALL NOT implement the full lists or maps modules — only functions actually imported by gleam_otp
- This brief SHALL NOT load OTP .beam files
- This brief SHALL NOT implement io:* or string:* functions (separate brief)

## Requirements

### R1: lists module stubs

Implement:
- `lists:map/2`: Apply function to each element of a list, return new list. The function argument is a BEAM fun (closure or export).
- `lists:foldr/3`: Right fold over a list with accumulator and function.
- `lists:reverse/1`: Reverse a list.

All list operations SHALL handle proper lists only. Improper lists SHALL return `badarg`.

**Acceptance:**
- lists:map(fun(X) -> X*2 end, [1,2,3]) returns [2,4,6]
- lists:foldr(fun(X, Acc) -> X+Acc end, 0, [1,2,3]) returns 6
- lists:reverse([1,2,3]) returns [3,2,1]
- Empty list input returns empty list
- Non-list input returns badarg

**Files:**
- create: crates/beamr/src/native/stdlib_stubs.rs
- modify: crates/beamr/src/native/mod.rs

**Checklist:**
- C150

### R2: maps module stubs

Implement:
- `maps:from_list/1`: Convert a list of `{Key, Value}` 2-tuples to a map.
- `maps:map/2`: Apply function to each value in a map, return new map.
- `maps:merge/2`: Merge two maps, second overrides first on key collision.
- `maps:remove/2`: Remove a key from a map, return new map.

**Acceptance:**
- maps:from_list([{a,1},{b,2}]) returns #{a => 1, b => 2}
- maps:map(fun(_K,V) -> V*2 end, #{a => 1}) returns #{a => 2}
- maps:merge(#{a => 1}, #{b => 2}) returns #{a => 1, b => 2}
- maps:merge(#{a => 1}, #{a => 2}) returns #{a => 2}
- maps:remove(a, #{a => 1, b => 2}) returns #{b => 2}

**Files:**
- create: crates/beamr/src/native/stdlib_stubs.rs

**Checklist:**
- C150

### R3: timer:sleep/1

`timer:sleep/1` SHALL accept an integer milliseconds and suspend the calling process for that duration. It SHALL return `ok`. This integrates with the scheduler's timer wheel. For the single-process CLI path, it can use `std::thread::sleep`.

**Acceptance:**
- timer:sleep(0) returns ok immediately
- timer:sleep(100) suspends for ~100ms then returns ok
- Non-integer argument returns badarg
- All stdlib stubs registered under their OTP module names

**Files:**
- create: crates/beamr/src/native/stdlib_stubs.rs

**Checklist:**
- C150

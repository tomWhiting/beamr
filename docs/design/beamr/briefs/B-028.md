# B-028: Stdlib stubs — lists, maps, timer as native BIFs

> **Cluster:** beamr
> **Depends on:** B-026
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer
> **Stories:**
> - S4 (Workflow Author, Writing Gleam Workflows) — As a workflow author, I want to call Rust-native operations (git, AST merge, dependency graph) from Gleam via registered functions so that I get zero-overhead access to the host's capabilities.
> - S9 (Meridian Runtime, Embedding beamr for Workflow Execution) — As the Meridian runtime, I want to register Rust functions as callable BIFs and NIFs before loading modules so that Gleam code can call Yggdrasil, git, and Meridian operations.

## Purpose

gleam_erlang_ffi and gleam_otp import OTP stdlib functions. Implement as native Rust stubs with correct semantics rather than loading OTP .beam files.

## Task

Implement lists:map/2, lists:foldr/3, lists:reverse/1, maps:from_list/1, maps:map/2, maps:merge/2, maps:remove/2, timer:sleep/1 in stdlib_stubs.rs.

## Requirements

### R1: lists module stubs

Implement lists:map/2, lists:foldr/3, lists:reverse/1 with correct semantics on proper lists.

**Acceptance:**
- lists:map(Fun, [1,2,3]) returns [2,4,6]
- lists:foldr(Fun, 0, [1,2,3]) returns 6
- lists:reverse([1,2,3]) returns [3,2,1]
- Empty list returns empty list
- Non-list returns badarg

**Files:**
- create: crates/beamr/src/native/stdlib_stubs.rs
- modify: crates/beamr/src/native/mod.rs

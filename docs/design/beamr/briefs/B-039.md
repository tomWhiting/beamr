# B-039: String, binary, and I/O BIFs — remaining sample workflow coverage

> **Cluster:** beamr
> **Depends on:** B-033
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

The sample workflow imports string operations (find, next_grapheme, pad, replace, slice), binary encoding (hex, base64), and I/O functions (put_chars, format) that are not yet implemented. These are needed for Gleam's string processing, output formatting, and binary encoding pipelines.

## Task

Implement 16 BIFs across the string, binary, base64, io, io_lib, and init modules.

## Requirements

### R1: string module BIFs (5 functions)

Register under string module: find/2, next_grapheme/1, pad/4, replace/4, slice/3.

**Acceptance:**
- string:find("hello world", "world") returns "world"
- string:find("hello", "xyz") returns nomatch
- string:pad("hi", 5, trailing, " ") returns "hi   "
- string:replace("aaa", "a", "b", all) returns "bbb"
- Tests cover each function

**Files:**
- modify: crates/beamr/src/native/stdlib_stubs/string_bifs.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R2: binary and base64 BIFs (4 functions)

binary:decode_hex/1, binary:encode_hex/1, base64:decode/1, base64:encode/2.

**Acceptance:**

- binary:decode_hex(<<"48454C4C4F">>) returns <<"HELLO">>
- base64:encode(<<"hello">>, standard) returns correct base64
- base64:decode(<<"aGVsbG8=">>) returns <<"hello">>
- Tests cover encode/decode round-trips

**Files:**

### R3: I/O BIFs with configurable output sink (5 functions)

Define IoSink trait with write(&self, bytes: &[u8]). Default NullSink (no-op), beamr-cli sets StdoutSink. Register io:put_chars/1, io:put_chars/2, io:format/3, io:setopts/2 (stub ok), io_lib:format/2.

## Boundaries

- SHALL NOT implement full Erlang format string parsing for io:format — support ~s, ~p, ~w, ~n only
- SHALL NOT implement io:setopts beyond returning ok (stub)
- SHALL route io:put_chars through a configurable output sink (trait/callback), not directly to stdout — default is no-op, beamr-cli sets stdout

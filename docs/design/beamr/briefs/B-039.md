# B-039: String, binary, and I/O BIFs — remaining sample workflow coverage

> **Cluster:** beamr
> **Depends on:** B-033 (stdlib stubs foundation)
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

The sample workflow imports string operations (find, next_grapheme, pad, replace, slice), binary encoding (hex, base64), and I/O functions (put_chars, format) that are not yet implemented. These are needed for Gleam's string processing, output formatting, and binary encoding pipelines.

## Task

Implement 16 BIFs across the string, binary, base64, io, io_lib, and init modules.

## Boundaries

- SHALL NOT implement full Erlang format string parsing for io:format — support ~s, ~p, ~w, ~n only
- SHALL NOT implement io:setopts beyond returning ok (stub)
- SHALL route io:put_chars through a configurable output sink (trait/callback), not directly to stdout — default is no-op, beamr-cli sets stdout
- SHALL stub uri_string functions as approximate implementations (not full RFC 3986)

## Requirements

### R1: string module BIFs (5 functions)

Register under `string` module:

| Function | Arity | Behaviour |
|----------|-------|-----------|
| find | 2 | Find first occurrence of pattern, return substring from match or nomatch |
| next_grapheme | 1 | Split first grapheme from string (byte-level for ASCII) |
| pad | 4 | Pad string to length with char and direction |
| replace | 4 | Replace occurrences with direction (leading/trailing/all) |
| slice | 3 | Extract substring by offset and length |

**Acceptance:**
- `string:find("hello world", "world")` returns `"world"`
- `string:find("hello", "xyz")` returns `nomatch`
- `string:pad("hi", 5, trailing, " ")` returns `"hi   "`
- `string:replace("aaa", "a", "b", all)` returns `"bbb"`
- Tests cover each function

**Files:**

**Checklist:** C150

### R2: binary and base64 BIFs (4 functions)

| Module | Function | Arity | Behaviour |
|--------|----------|-------|-----------|
| binary | decode_hex | 1 | Hex string to bytes |
| binary | encode_hex | 1 | Bytes to uppercase hex string |
| base64 | decode | 1 | Base64 string to bytes |
| base64 | encode | 2 | Bytes to base64 string (mode atom) |

**Acceptance:**
- `binary:encode_hex(<<"hello">>)` returns `<<"68656C6C6F">>`

- `base64:decode(<<"aGVsbG8=">>)` returns `<<"hello">>`
- Tests cover encode/decode round-trips

### R3: I/O BIFs with configurable output sink (5 functions)

Define an `IoSink` trait with `write(&self, bytes: &[u8])`. The scheduler or runtime holds an `Arc<dyn IoSink>`. Default is `NullSink` (no-op). beamr-cli sets `StdoutSink`.

| Module | Function | Arity | Behaviour |
|--------|----------|-------|-----------|
| io | put_chars | 1 | Write binary to output sink |
| io | put_chars | 2 | Write binary to device (ignore device, use sink) |
| io | format | 3 | Format string with args, write to sink |
| io | setopts | 2 | Stub — return ok |
| io_lib | format | 2 | Format string with args, return as binary |

**Acceptance:**
- `io:put_chars(<<"hello">>)` writes to configured sink
- `io_lib:format("~s ~s", ["hello", "world"])` returns `<<"hello world">>`
- Default NullSink produces no output
- beamr-cli StdoutSink writes to stdout
- Tests verify sink wiring

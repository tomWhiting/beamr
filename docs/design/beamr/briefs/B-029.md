# B-029: Utility stubs — logger, unicode, sys, gleam_stdlib identity

> **Cluster:** beamr
> **Depends on:** B-026
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer
> **Stories:**
> - S4 (Workflow Author) — As a workflow author, I want to call Rust-native operations from Gleam via registered functions.

## Purpose

gleam_otp actor and supervisor modules import utility functions from OTP modules: `logger:warning/2` for diagnostics, `sys:debug_options/1` for OTP debug traces, `unicode:characters_to_list/1` and `unicode:characters_to_binary/1` for string conversion, and `gleam_stdlib:identity/1` as a passthrough function. These are all simple to stub with correct semantics.

## Task

Implement utility stubs in `crates/beamr/src/native/stdlib_stubs.rs` (created in B-028). Register under their OTP module names.

## Boundaries

- This brief SHALL NOT implement the full logger, sys, or unicode modules
- This brief SHALL NOT implement OTP debug tracing (sys:debug_options returns an empty list)
- This brief SHALL NOT implement io:format or io:fwrite

## Requirements

### R1: logger:warning/2 and gleam_stdlib:identity/1

`logger:warning/2` SHALL accept (Format, Args) and write a warning message to stderr (or the tracing system if configured). For now, format the message using a simple string interpolation and write to stderr. `gleam_stdlib:identity/1` SHALL return its argument unchanged.

**Acceptance:**
- logger:warning("msg: ~p", [detail]) writes to stderr
- logger:warning with non-string format is a no-op (not crash)
- gleam_stdlib:identity(X) returns X for any term

**Files:**
- modify: crates/beamr/src/native/stdlib_stubs.rs

**Checklist:**
- C150

### R2: unicode conversions

`unicode:characters_to_list/1` SHALL convert a binary or iolist to a list of Unicode code points. `unicode:characters_to_binary/1` SHALL convert a list of code points or an iolist to a UTF-8 binary. Since Gleam strings are already UTF-8 binaries, the binary→binary case is a passthrough.

**Acceptance:**
- unicode:characters_to_binary(<<"hello">>) returns <<"hello">>
- unicode:characters_to_list(<<"abc">>) returns [97, 98, 99]
- unicode:characters_to_binary([104, 105]) returns <<"hi">>
- Invalid input returns {error, Encoded, Rest} tuple (BEAM semantics)

**Files:**
- modify: crates/beamr/src/native/stdlib_stubs.rs

**Checklist:**
- C150

### R3: sys:debug_options/1

`sys:debug_options/1` SHALL accept an options list and return a debug state. For beamr, return an empty list `[]` (no OTP debug tracing). This satisfies gleam_otp actor's init path which calls `sys:debug_options([])`.

**Acceptance:**
- sys:debug_options([]) returns []
- sys:debug_options(any_list) returns [] (no-op stub)
- All utility stubs registered under their OTP module names

**Files:**
- modify: crates/beamr/src/native/stdlib_stubs.rs

**Checklist:**
- C150

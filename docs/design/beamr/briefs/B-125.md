# B-125: Process groups — pg (scope-based groups, cross-node propagation)

> **Cluster:** beamr
> **Depends on:** B-123
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

The pg module provides named process groups that span the cluster. A process can join one or more named groups; other processes can query group membership. Group changes propagate to all connected nodes. pg replaced the older pg2 module in OTP 23+.

## Task

Implement the pg module with scope-based groups and cross-node membership propagation.

## Requirements

### R1: Local process groups

Implement local group membership: join, leave, get_members, get_local_members.

**Acceptance:**
- pg:join(Group, Pid) adds Pid to the named group
- pg:leave(Group, Pid) removes Pid from the group
- pg:get_members(Group) returns all members (local and remote)
- pg:get_local_members(Group) returns only local members
- A process can be in multiple groups
- A process leaving or dying is automatically removed from all groups
- Test: join, get_members, leave, verify membership

**Files:**
- create: crates/beamr/src/distribution/pg.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R2: Cross-node group propagation

Group membership changes propagate to all connected nodes.

**Acceptance:**
- When a process joins a group, the join is broadcast to all connected nodes
- Remote nodes update their group membership tables
- When a process leaves or dies, the leave is broadcast
- Node disconnection removes all members from the disconnected node
- Test: process on A joins group, node B sees the member in get_members

**Files:**
- modify: crates/beamr/src/distribution/pg.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R3: Scoped groups

pg supports scopes — isolated group namespaces. The default scope is 'pg'.

**Acceptance:**
- pg:start_link(Scope) creates a named scope
- Groups in different scopes are independent
- Default scope 'pg' is available without explicit start
- Test: two scopes, same group name, different members

**Files:**
- modify: crates/beamr/src/distribution/pg.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Boundaries

- SHALL NOT implement pg2 (deprecated predecessor) — only pg (OTP 23+)
- SHALL NOT implement pg monitors (monitoring group membership changes) in this brief
- SHALL handle network partitions by removing unreachable members

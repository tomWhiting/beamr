# B-031: gleam_erlang_ffi — process utilities

> **Cluster:** beamr
> **Depends on:** B-027, B-030
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer
> **Stories:**
> - S2 (Workflow Author, Writing Gleam Workflows) — As a workflow author, I want each workflow step to run in its own isolated process so that a crash in one step does not corrupt the state of another.
> - S6 (Workflow Author, Writing Gleam Workflows) — As a workflow author, I want to define supervision trees over my workflow steps so that failed steps are automatically restarted according to a declared strategy.

## Purpose

gleam_erlang_ffi process utilities are thin wrappers around existing BIFs. Implement as native Rust under gleam_erlang_ffi MFAs.

## Task

Implement trap_exits, link, demonitor, sleep, flush_messages, register/unregister_process, process_named, pid_from_dynamic.

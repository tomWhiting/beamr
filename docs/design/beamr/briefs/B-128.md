# B-128: erlang BIFs batch 3 — system, memory, and port stubs

> **Cluster:** beamr
> **Depends on:** B-081
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

OTP modules query system state via erlang:statistics/1, erlang:memory/0, erlang:ports/0, and various system introspection BIFs. Most can be stubbed with reasonable defaults — beamr doesn't have ports or full statistics, but returning empty lists or zero values prevents import failures.

## Task

Register system introspection BIF stubs that return sensible defaults.

## Requirements

### R1: erlang:statistics/1 stubs

Return basic statistics for commonly queried items.

**Acceptance:**
- statistics(wall_clock) returns {TotalMillis, SinceLastMillis}
- statistics(runtime) returns {TotalMillis, SinceLastMillis}
- statistics(reductions) returns {Total, SinceLast}
- Unknown stat item returns badarg

**Files:**
- modify: crates/beamr/src/native/system_info_bifs.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R2: erlang:memory/0 and erlang:memory/1

Return approximate memory usage.

**Acceptance:**
- memory() returns proplist with total, processes, system, atom, binary keys
- memory(total) returns total memory in bytes
- Values are approximate (sum of heap sizes + atom table + binary heap)

**Files:**
- modify: crates/beamr/src/native/system_info_bifs.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R3: Port stubs

Stub port-related BIFs that OTP queries during startup.

**Acceptance:**
- erlang:ports() returns [] (no ports in beamr)
- erlang:port_info/1 returns undefined
- erlang:open_port/2 returns badarg (ports not supported)
- These stubs prevent import failures without implementing the port subsystem

**Files:**
- modify: crates/beamr/src/native/system_info_bifs.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Boundaries

- SHALL NOT implement the port subsystem — stubs only
- SHALL NOT implement full statistics tracking — approximate values from available data
- SHALL return sensible defaults that don't crash OTP code

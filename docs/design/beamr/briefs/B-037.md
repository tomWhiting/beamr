# B-037: Core erlang BIFs — arithmetic, type conversion, bitwise, and math

> **Cluster:** beamr
> **Depends on:** B-033 (stdlib stubs foundation)
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

The sample workflow's dependency chain imports 32 functions from erlang, math, and rand that are not yet implemented. These cover type conversions (integer/float/atom to binary), bitwise operations (band, bor, bxor, bsl, bsr, bnot), math functions (ceil, floor, pow, log, exp), and rounding (round, trunc). Without these, any Gleam workflow using number formatting, bitwise flags, or math operations fails at runtime.

## Task

Implement 32 BIFs across the erlang, math, and rand modules as native Rust stubs.

## Boundaries

- SHALL NOT implement arbitrary-precision integer arithmetic (SmallInt range is sufficient)
- SHALL NOT implement erlang:float_to_binary with full formatting options (basic to_string is sufficient)
- SHALL return badarg for unsupported argument types rather than panicking

## Requirements

### R1: erlang type conversion BIFs (12 functions)

Register under `erlang` module:

| Function | Arity | Behaviour |
|----------|-------|-----------|
| atom_to_binary | 1 | Atom name as binary |
| binary_to_float | 1 | Parse binary string to float term |
| binary_to_integer | 1 | Parse binary string to integer term |
| binary_to_integer | 2 | Parse with radix (2-36) |
| float | 1 | Convert integer to float |
| integer_to_binary | 1 | Integer to decimal binary string |
| integer_to_binary | 2 | Integer to binary string with radix |
| integer_to_list | 1 | Integer to list of character codes |
| iolist_to_binary | 1 | Flatten iolist to single binary |
| list_to_bitstring | 1 | Convert list of bytes to binary |
| list_to_tuple | 1 | Convert proper list to tuple |
| tuple_to_list | 1 | Convert tuple to proper list |

**Acceptance:**
- `atom_to_binary(ok)` returns `<<"ok">>`
- `binary_to_integer(<<"42">>)` returns `42`
- `binary_to_integer(<<"FF">>, 16)` returns `255`
- `integer_to_binary(42)` returns `<<"42">>`
- `list_to_tuple([1, 2, 3])` returns `{1, 2, 3}`
- `tuple_to_list({1, 2, 3})` returns `[1, 2, 3]`
- Non-matching types return badarg
- Tests cover each function

**Files:**

- create: crates/beamr/src/native/stdlib_stubs/type_conversion_bifs.rs

**Checklist:** C150

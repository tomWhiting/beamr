# B-121: Remote monitors — MONITOR_P/DEMONITOR_P/MONITOR_P_EXIT

> **Cluster:** beamr
> **Depends on:** B-119
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

Cross-node monitors use MONITOR_P (establish), DEMONITOR_P (remove), and MONITOR_P_EXIT (DOWN notification) control messages. When a monitored process on node B exits, node B sends MONITOR_P_EXIT to node A, which delivers the DOWN message to the monitoring process.

## Task

Implement cross-node monitor control messages.

## Requirements

### R1: Remote monitor establishment

When monitor(process, RemotePid) is called, send MONITOR_P to the remote node.

**Acceptance:**
- Outbound: monitor on remote PID sends MONITOR_P control with reference
- Inbound: receiving MONITOR_P registers the monitor on the local process
- If the target process is already dead, immediately send MONITOR_P_EXIT back
- Test: monitor remote process, verify monitor established

**Files:**
- modify: crates/beamr/src/distribution/control.rs
- modify: crates/beamr/src/native/process_bifs/mod.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R2: Remote DOWN delivery

When a monitored process exits, deliver MONITOR_P_EXIT to all remote monitors.

**Acceptance:**
- Process exit triggers MONITOR_P_EXIT for each remote monitor
- Remote node receives MONITOR_P_EXIT and delivers DOWN message to the monitoring process
- DOWN message contains correct Ref, Pid, and Reason
- Test: monitor remote process, it exits, verify DOWN received

**Files:**
- modify: crates/beamr/src/distribution/control.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R3: Remote demonitor

DEMONITOR_P removes a cross-node monitor.

**Acceptance:**
- demonitor(Ref) for a remote monitor sends DEMONITOR_P to the remote node
- Remote node removes the monitor registration
- No DOWN sent after demonitor even if the process exits
- Test: monitor, demonitor, process exits, no DOWN received

**Files:**
- modify: crates/beamr/src/distribution/control.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Boundaries

- SHALL NOT implement monitor by name across nodes — only by PID
- SHALL handle node-down as a special DOWN reason (noconnection)

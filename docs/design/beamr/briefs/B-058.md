# B-058: swap opcode — register swap (OTP 27+)

> **Cluster:** beamr
> **Depends on:** B-007 (interpreter loop and basic dispatch)
> **Checklist:**
> - C62 — Move instruction copies terms between registers

## Purpose

The `swap` opcode (opcode 169) atomically exchanges the values of two registers. The OTP 27+ compiler emits `swap` frequently as an optimization — it replaces a 3-instruction `move` sequence (A→tmp, B→A, tmp→B) with a single instruction. Without `swap`, any code compiled with OTP 27+ hits `UnsupportedOpcode(169)`. Since beamr targets OTP 27 bytecode, this is critical.

## Task

Wire the existing `Instruction::Swap { left, right }` variant (already decoded by the loader) into the opcode dispatch table and implement the handler in `core.rs`.

## Boundaries

- SHALL NOT allocate heap space — swap copies 64-bit Term values only (same as move)
- SHALL NOT add a temporary register — read both values, then write both
- Note: core.rs is at 583 lines (over 500-line guideline). swap adds ~15 lines. Full core.rs split is a separate concern.

## Requirements

### R1: Opcode dispatch wiring

Add `Instruction::Swap` to the dispatch table in `opcodes/mod.rs`, routing to a handler in `core.rs`.

**Acceptance:**
- `Instruction::Swap { left, right }` is matched in the dispatch table
- Dispatch routes to `core::swap()`
- No `UnsupportedOpcode(169)` error when executing modules containing `swap`

**Files:**
- modify: crates/beamr/src/interpreter/opcodes/mod.rs

**Checklist:** C62

### R2: swap handler implementation

Implement `swap` in `core.rs`. The instruction reads both operands, then writes each value to the other's location.

**Acceptance:**
- `swap(process, module, left, right)` exchanges the values of two registers
- Both operands are read via `core::read_term()` before either write
- Both values are written via `core::write_term()` after both reads (no clobber)
- Works for X-X, Y-Y, and X-Y register combinations
- Test: X0=42, X1=99. swap(X0, X1). X0=99, X1=42.
- Test: Y0=atom(hello), X0=atom(world). swap(Y0, X0). Y0=atom(world), X0=atom(hello).
- Test: swap(X0, X0) is a no-op (same register)

**Files:**
- modify: crates/beamr/src/interpreter/opcodes/core.rs

**Checklist:** C62

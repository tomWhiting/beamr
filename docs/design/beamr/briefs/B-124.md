# B-124: Global name registration — global:register_name and conflict resolution

> **Cluster:** beamr
> **Depends on:** B-123
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

The global module provides cluster-wide name registration — a name atom maps to exactly one PID across all connected nodes. Registration uses a leader election protocol to resolve conflicts when the same name is registered on multiple nodes simultaneously.

## Task

Implement global name registration with a simple conflict resolution strategy.

## Requirements

### R1: Global name registry

A distributed name registry that maps atom names to PIDs across the cluster.

**Acceptance:**
- global:register_name(Name, Pid) registers the name cluster-wide
- global:whereis_name(Name) returns the PID (local or remote) or undefined
- global:unregister_name(Name) removes the registration
- Registration propagates to all connected nodes
- Test: register on node A, whereis on node B returns correct PID

**Files:**
- create: crates/beamr/src/distribution/global.rs
- modify: crates/beamr/src/native/ets_bifs.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R2: Conflict resolution

When the same name is registered on two nodes simultaneously (network partition recovery), resolve the conflict.

**Acceptance:**
- Default resolution: the registration with the lower node name wins (deterministic)
- Losing PID is notified (exit signal or callback)
- Custom resolve function support via global:register_name/3
- Test: register same name on two nodes, reconnect, verify one registration survives

**Files:**
- modify: crates/beamr/src/distribution/global.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Boundaries

- SHALL NOT implement the full global lock protocol — use a simplified single-leader approach
- SHALL NOT implement global:trans (global transactions)
- SHALL handle node disconnection by cleaning up registrations for PIDs on the disconnected node

# B-095: ETS iteration and folding

> **Cluster:** beamr
> **Depends on:** B-094
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

ETS tables need to be iterable — `tab2list/1` dumps all entries, `foldl/3` accumulates over entries, and `first/next/last/prev` provide cursor-based traversal for ordered_set tables. OTP uses these for diagnostics, serialization, and ordered iteration.

## Task

Implement iteration and folding BIFs for all table types.

## Requirements

### R1: ets:tab2list/1

ets:tab2list/1

**Acceptance:**
- Returns a list of all tuples in the table
- Set/bag: unordered. Ordered_set: sorted by key.
- Empty table returns empty list
- Test: insert 3 entries, tab2list returns all 3

**Files:**
- modify: crates/beamr/src/native/ets_bifs.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R2: ets:foldl/3

ets:foldl/3

**Acceptance:**
- `ets:foldl(Fun, Acc0, Tab)` folds over all entries with a function
- Fun receives (Element, AccIn) and returns AccOut
- Uses the trampoline calling convention (B-038 R3) for closure invocation
- Test: foldl to sum values in a table

**Files:**
- modify: crates/beamr/src/native/ets_bifs.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R3: ets:first/1, ets:next/2, ets:last/1, ets:prev/2

ets:first/1, ets:next/2, ets:last/1, ets:prev/2

**Acceptance:**
- `first(Tab)` returns smallest key (ordered_set) or any key (set/bag)
- `next(Tab, Key)` returns next key, or `'$end_of_table'` atom at end
- `last(Tab)` returns largest key (ordered_set)
- `prev(Tab, Key)` returns previous key
- For non-ordered tables: first/next provide arbitrary-but-complete traversal
- Test: ordered_set traversal via first/next visits all keys in order

**Files:**
- modify: crates/beamr/src/native/ets_bifs.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Boundaries

- SHALL NOT implement safe iteration under concurrent modification — iteration sees a snapshot (or documents the caveat)
- SHALL NOT implement ets:match_object or ets:select — those need match specs (B-097)
- SHALL NOT implement continuation-based iteration in this brief

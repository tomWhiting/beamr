# B-036: beamr-meridian crate scaffold — MeridianRuntime, NIF wiring, run_workflow

> **Cluster:** beamr-meridian
> **Depends on:** B-033, B-035
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

The beamr-meridian crate is the integration layer between beamr (standalone BEAM VM) and Meridian (workflow executor). It creates a beamr runtime, registers workflow NIFs under meridian_ffi, loads compiled Gleam workflows, executes them, and converts the results for the Meridian orchestrator. This crate lives in the yggdrasil repo at crates/beamr-meridian/ with beamr as a git dependency.

## Task

Scaffold the beamr-meridian crate with the MeridianRuntime struct, NIF registrations for all workflow operations, the run_workflow entry point, and a step runner bridge for dispatching AI agent steps.

### R3: run_workflow entry point

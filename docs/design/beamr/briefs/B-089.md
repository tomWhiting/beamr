# B-089: term_to_iovec — scatter-gather encoding

> **Cluster:** beamr
> **Depends on:** B-086 (ETF encoder)
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

`term_to_binary/1` (B-086) encodes a term into a single contiguous binary, copying all data — including large binaries embedded in the term — into the output buffer. For terms containing multi-megabyte binaries, this copy is expensive and wasteful when the output is immediately sent over a socket (which can scatter-gather from multiple buffers).

`term_to_iovec/1` returns an IO list (a list of binaries) instead of a single binary. The key optimization: large binaries in the source term are *referenced* directly in the output list, not copied. The ETF framing (tags, headers, lengths) is in small binaries interleaved with the large binary references.

When the IO list is sent via a port or socket, the runtime can use `writev()` / scatter-gather I/O to send all segments without copying. This is how OTP's distribution protocol efficiently transfers large messages.

## Boundaries

- This brief SHALL implement `term_to_iovec/1` and `term_to_iovec/2`
- This brief SHALL reference (not copy) binaries above a size threshold
- This brief SHALL produce output that, when concatenated, equals `term_to_binary` output byte-for-byte
- This brief SHALL NOT implement IO list flattening or sending — that's the port/socket layer's concern

## Requirements

### R1: Scatter-gather encoder

Extend the ETF encoder to produce a list of byte segments instead of a single buffer.

**Acceptance:**
- New function: `etf::encode::encode_term_iovec(term: Term, atom_table: &AtomTable) -> Result<Vec<IoSegment>, EncodeError>`
- `IoSegment` enum:
  - `Owned(Vec<u8>)` — ETF framing bytes (tags, headers, lengths) that the encoder generates
  - `Reference(Term)` — a binary Term referenced directly (not copied). The Term must be a valid binary.
- Size threshold: binaries >= 64 bytes are referenced; smaller binaries are copied into the framing buffer (64 bytes matches OTP's `ERL_ONHEAP_BIN_LIMIT`)
- The version byte (131), all tags, headers, and small data go into Owned segments
- Large binaries in tuples, lists, maps emit: `Owned([BINARY_EXT tag, 4-byte length])` + `Reference(binary_term)` — the framing and the data are separate segments
- When all segments are concatenated in order, the result is byte-identical to `encode_term()`
- Test: term with small binary (< 64 bytes) produces single Owned segment (same as term_to_binary)
- Test: term with large binary (>= 64 bytes) produces Owned + Reference segments
- Test: concatenation of all segments equals term_to_binary output

**Files:**
- modify: crates/beamr/src/etf/encode.rs (IoSegment, encode_term_iovec)

**Checklist:** C150

### R2: erlang:term_to_iovec/1,2 BIFs

Register the BIFs and build the IO list result on the process heap.

**Acceptance:**
- `erlang:term_to_iovec/1` registered as BIF
- `erlang:term_to_iovec/2` registered as BIF (options same as term_to_binary/2 — compression from B-088 applies to the framing segments only, not referenced binaries)
- Calls `encode_term_iovec`, then builds an Erlang list on the process heap:
  - `Owned(bytes)` allocates as binary term on heap
  - `Reference(term)` uses the binary term directly (no copy)
- Returns the list of binaries
- Test: `term_to_iovec({data, LargeBinary})` returns a list with >= 2 elements
- Test: `iolist_to_binary(term_to_iovec(X))` would equal `term_to_binary(X)` (semantic equivalence)

**Files:**
- modify: crates/beamr/src/native/etf_bifs.rs (BIF registration)

**Checklist:** C150

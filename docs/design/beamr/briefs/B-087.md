# B-087: binary_to_term decoder ‚Äî ETF to runtime Terms on process heap

> **Cluster:** beamr
> **Depends on:** B-086 (shared etf/ module and tag constants)
> **Checklist:**
> - C150 ‚Äî BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

beamr has an ETF decoder in the loader pipeline (`loader/decode/etf.rs`) that parses ETF-encoded literals from BEAM files into a `Literal` enum, which is then materialized into heap terms by the constant pool. This decoder is loader-specific: it reads from a `BoundedCursor`, produces intermediate `Literal` values, and allocates on a module-level constant pool ‚Äî not a process heap.

The `erlang:binary_to_term/1` BIF needs a different path: take an arbitrary binary term at runtime, parse the ETF bytes, and allocate the resulting term directly on the calling process's heap. This is the runtime decoder, complementing B-086's encoder to close the serialization round-trip.

The runtime decoder writes Terms directly to the process heap (using existing boxed writers ‚Äî `write_tuple`, `write_cons`, `write_float`, etc.) without the intermediate `Literal` representation. This avoids double allocation and is the standard approach for runtime deserialization.

## Boundaries

- This brief SHALL implement a runtime ETF decoder that allocates on the process heap
- This brief SHALL reuse the tag constants from `etf/tags.rs` (B-086)
- This brief SHALL include a decode budget for safety (depth, size, atom limits)
- This brief SHALL register `erlang:binary_to_term/1` as a BIF
- This brief SHALL NOT implement safe mode or options (B-088)
- This brief SHALL NOT reuse the loader's `Literal`-based decode path (different allocation strategy)

## Requirements

### R1: Runtime decode budget

Adapt the loader's budget concept for runtime use, protecting against malicious or malformed input.

**Acceptance:**
- `etf/decode.rs` with `RuntimeDecodeBudget` struct:
  - `max_depth: usize` ‚Äî maximum nesting depth (default 256, matching loader)
  - `max_heap_words: usize` ‚Äî maximum heap words to allocate (default: remaining heap capacity of the calling process)
  - `current_depth: usize` ‚Äî tracks current recursion depth
  - `heap_words_used: usize` ‚Äî tracks allocations
- `descend()` / `ascend()` methods for depth tracking, return error on depth exceeded
- `charge_heap(words: usize)` method, return error on budget exceeded
- Error type: `DecodeError` with variants `DepthExceeded`, `HeapBudgetExceeded`, `InvalidTag(u8)`, `Truncated`, `InvalidAtom`, `InvalidUtf8`

**Files:**
- create: crates/beamr/src/etf/decode.rs

**Checklist:** C150

### R2: Core runtime decoder ‚Äî decode_term function

Implement the main decode function that reads ETF bytes and allocates Terms on the process heap.

**Acceptance:**
- `pub fn decode_term(bytes: &[u8], process: &mut Process, atom_table: &AtomTable) -> Result<Term, DecodeError>`
- Validates version byte (131) at the start
- Tag dispatch matching the same 14+ tags as the loader decoder, but writing directly to the process heap:
  - `SMALL_INTEGER_EXT` (97) ‚Üí `Term::small_int(value)`
  - `INTEGER_EXT` (98) ‚Üí `Term::small_int(i32 as i64)`
  - `NEW_FLOAT_EXT` (70) ‚Üí `write_float(heap, f64)` ‚Üí boxed Term
  - `ATOM_UTF8_EXT` (118), `SMALL_ATOM_UTF8_EXT` (119), `ATOM_EXT` (100) ‚Üí `atom_table.intern(name)` ‚Üí `Term::atom(atom)`
  - `SMALL_TUPLE_EXT` (104), `LARGE_TUPLE_EXT` (105) ‚Üí recursively decode elements, `write_tuple(heap, &elements)`
  - `NIL_EXT` (106) ‚Üí `Term::NIL`
  - `STRING_EXT` (107) ‚Üí build list of small integers (byte values) on heap via cons cells
  - `LIST_EXT` (108) ‚Üí recursively decode elements and tail, build cons list on heap
  - `BINARY_EXT` (109) ‚Üí `write_binary(heap, &bytes)`
  - `SMALL_BIG_EXT` (110), `LARGE_BIG_EXT` (111) ‚Üí `write_bigint(heap, sign, &limbs)` or `Term::small_int()` if fits
  - `MAP_EXT` (116) ‚Üí decode key-value pairs, `write_map(heap, &keys, &values)`
  - `EXPORT_EXT` (113) ‚Üí decode module/function/arity atoms, build `{Module, Function, Arity}` tuple
  - `NEW_PID_EXT` (88), `PID_EXT` (103) ‚Üí `Term::pid(id)` (extract local PID for local node, or build external PID term)
  - `NEWER_REFERENCE_EXT` (90), `REFERENCE_EXT` (114) ‚Üí `write_reference(heap, id)`
- Budget checks: charge heap for each allocation, descend/ascend for recursion
- Test: decode `[131, 97, 42]` ‚Üí Term::small_int(42)
- Test: decode encoded tuple `{ok, 42}` ‚Üí 2-tuple on heap
- Test: decode encoded list `[1, 2, 3]` ‚Üí cons list on heap
- Test: round-trip: encode a term with B-086's encoder, decode it back, verify equality

### R3: erlang:binary_to_term/1 BIF

Register the BIF and wire it to the decoder.

**Acceptance:**
- `erlang:binary_to_term/1` registered in BIF registry
- Takes a binary term as argument
- Extracts bytes via `Binary::new(arg)?.as_bytes()`
- Calls `etf::decode::decode_term(bytes, process, atom_table)`
- Returns the decoded term
- On non-binary argument, returns `ExecError::Badarg`
- On `DecodeError::*`, returns `ExecError::Badarg` (matching BEAM semantics ‚Äî invalid ETF is a badarg)
- Test: `binary_to_term(term_to_binary(X))` round-trips for atoms, integers, tuples, lists, maps, binaries

# B-094: Core ETS BIFs — new, insert, lookup, delete, member, info

> **Cluster:** beamr
> **Depends on:** B-091, B-093, B-092
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

This brief registers the fundamental ETS BIFs that create tables, insert data, look up entries, and query table metadata. Every OTP module that uses ETS calls these functions. Without them, `ets:new`, `ets:insert`, and `ets:lookup` are unresolved imports.

## Task

Implement 8 core ETS BIFs in a new `native/ets_bifs.rs` file.

## Requirements

### R1: ets:new/2

Register `ets:new/2`. Accepts (Name, Options). Returns table ID (or Name atom for named tables).

**Acceptance:**
- Options list parsed: type (set/ordered_set/bag/duplicate_bag, default set), access (public/protected/private, default protected), named_table, {keypos, N}
- Creates table in registry, sets calling process as owner
- Named tables: name atom is returned and used for subsequent lookups
- Unnamed tables: integer table ID returned
- Duplicate named table returns badarg
- Test: `ets:new(my_table, [set, public, named_table])` creates a public set table

**Files:**
- create: crates/beamr/src/native/ets_bifs.rs
- modify: crates/beamr/src/native/mod.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R2: ets:insert/2 and ets:lookup/2

ets:insert/2 and ets:lookup/2

**Acceptance:**
- `ets:insert(Tab, Tuple)` inserts a single tuple, returns true
- `ets:insert(Tab, [Tuple1, Tuple2, ...])` inserts multiple tuples, returns true
- `ets:lookup(Tab, Key)` returns list of matching tuples (empty list if none)
- Access control enforced (protection level)
- Test: insert and lookup round-trip for set, bag, ordered_set

**Files:**
- modify: crates/beamr/src/native/ets_bifs.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R3: ets:delete/1, ets:delete/2, ets:member/2

ets:delete/1, ets:delete/2, ets:member/2

**Acceptance:**
- `ets:delete(Tab)` deletes the entire table, returns true. Only owner can delete.
- `ets:delete(Tab, Key)` deletes all entries with key, returns true
- `ets:member(Tab, Key)` returns true/false
- Test: insert, member returns true, delete key, member returns false

**Files:**
- modify: crates/beamr/src/native/ets_bifs.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R4: ets:info/1, ets:info/2

ets:info/1, ets:info/2

**Acceptance:**
- `ets:info(Tab)` returns a proplist of all table metadata: name, type, size, keypos, protection, owner, memory
- `ets:info(Tab, Item)` returns a single info item value
- `size` returns current entry count
- `memory` returns approximate word count
- Non-existent table returns `undefined`
- Test: create table, insert entries, verify info(size) matches

**Files:**
- modify: crates/beamr/src/native/ets_bifs.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

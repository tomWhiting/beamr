# B-086: term_to_binary encoder — all term types to ETF wire format

> **Cluster:** beamr
> **Depends on:** (none — uses existing term and boxed infrastructure)
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

BEAM's External Term Format (ETF) is the canonical wire encoding for Erlang terms. `term_to_binary/1` converts any runtime term to a binary containing the ETF representation. It's used for:
- Distribution protocol (node-to-node message passing)
- Persistent storage (ETS dets tables, mnesia)
- Message serialization in ports and NIFs
- `phash2` and term comparison across nodes

beamr has a battle-tested ETF *decoder* in `loader/decode/etf.rs` (14 tags, budget system, zlib decompression) but no *encoder*. Without encoding, beamr processes cannot serialize terms for distribution, storage, or cross-boundary communication.

This brief adds the encoding direction: given a runtime `Term`, produce the ETF binary representation. The output is a `Vec<u8>` that gets allocated as a binary term on the process heap.

## Boundaries

- This brief SHALL encode all runtime term types to ETF
- This brief SHALL create a shared `etf/tags.rs` module for ETF tag constants used by both encoder and loader decoder
- This brief SHALL register `erlang:term_to_binary/1` as a BIF
- This brief SHALL NOT implement compression (B-088)
- This brief SHALL NOT implement options (B-088)
- This brief SHALL NOT implement scatter-gather / iovec output (B-089)

### R2: Core encoder — encode_term function

Implement the main encoding function that dispatches on term type and writes ETF bytes.

**Acceptance:**
- `etf/encode.rs` with `pub fn encode_term(term: Term, atom_table: &AtomTable) -> Result<Vec<u8>, EncodeError>`
- Output begins with version byte (131)
- Encoding selects the most compact tag for each value:
  - **Small integer** (0–255): `SMALL_INTEGER_EXT` (1+1 bytes)
  - **Integer** (i32 range): `INTEGER_EXT` (1+4 bytes)
  - **Large integer** (outside i32): `SMALL_BIG_EXT` or `LARGE_BIG_EXT`
  - **Float**: `NEW_FLOAT_EXT` (1+8 bytes, IEEE 754)
  - **Atom**: `SMALL_ATOM_UTF8_EXT` if name ≤ 255 bytes, else `ATOM_UTF8_EXT`. Resolves atom name via `atom_table.resolve(atom)`
  - **NIL**: `NIL_EXT` (1 byte)
  - **PID**: `NEW_PID_EXT` with node atom `nonode@nohost`, creation 0 (local node representation for now)
  - **Reference**: `NEWER_REFERENCE_EXT` with node atom, creation 0
- New error type: `EncodeError` with variants `UnsupportedTerm`, `AtomResolveFailed`
- Test: encode small integer 42 → `[131, 97, 42]`
- Test: encode integer 1000 → `[131, 98, 0, 0, 3, 232]`
- Test: encode float 3.14 → `[131, 70, <8 IEEE bytes>]`
- Test: encode atom 'ok' → `[131, 119, 2, 111, 107]`
- Test: encode NIL → `[131, 106]`

**Files:**
- create: crates/beamr/src/etf/encode.rs

**Checklist:** C150

### R3: Compound term encoding — tuples, lists, maps, binaries

Extend the encoder for heap-allocated compound terms.

**Acceptance:**
- **Tuple**: `SMALL_TUPLE_EXT` for arity ≤ 255, `LARGE_TUPLE_EXT` for larger. Recursively encode each element.
- **List (cons)**: Walk the cons chain. If all elements are bytes (0–255) and tail is NIL, encode as `STRING_EXT` (optimization). Otherwise, encode as `LIST_EXT` with recursive elements and explicit tail encoding.
- **Binary**: `BINARY_EXT` — 4-byte length + raw bytes. Read bytes via existing `Binary::as_bytes()`.
- **Map**: `MAP_EXT` — 4-byte size + key-value pairs. Keys and values recursively encoded.
- **BigInt**: `SMALL_BIG_EXT` or `LARGE_BIG_EXT` — sign byte + little-endian limbs. Read from the existing BigInt heap layout.
- **Closure**: `EXPORT_EXT` for exported funs (module:function/arity). For captured closures, return `EncodeError::UnsupportedTerm` (BEAM's NEW_FUN_EXT is complex and can be deferred).
- Depth limit: recursive encoding limited to 256 nesting levels (matching the decode budget's MAX_ETF_DEPTH). Exceeding returns `EncodeError::TooDeep`.
- Test: encode `{ok, 42}` → SMALL_TUPLE_EXT with ATOM + SMALL_INTEGER
- Test: encode `[1, 2, 3]` → LIST_EXT with 3 integers + NIL tail
- Test: encode byte list `[72, 101, 108, 108, 111]` → STRING_EXT optimization
- Test: encode `#{a => 1}` → MAP_EXT
- Test: encode binary `<<1,2,3>>` → BINARY_EXT
- Test: deeply nested term (>256 levels) returns TooDeep

**Files:**
- modify: crates/beamr/src/etf/encode.rs

**Checklist:** C150

### R4: erlang:term_to_binary/1 BIF

Register the BIF and wire it to the encoder.

**Acceptance:**
- `erlang:term_to_binary/1` registered in BIF registry
- Takes any term as argument
- Calls `etf::encode::encode_term(arg, atom_table)`
- Allocates the resulting `Vec<u8>` as a binary term on the process heap via `write_binary(heap, &bytes)`
- Returns the binary term
- On `EncodeError::UnsupportedTerm`, returns `ExecError::Badarg`
- Test: from BIF call, `term_to_binary({ok, 42})` returns a binary containing valid ETF

**Files:**
- create: crates/beamr/src/native/etf_bifs.rs
- modify: crates/beamr/src/native/mod.rs (registration)

**Checklist:** C150

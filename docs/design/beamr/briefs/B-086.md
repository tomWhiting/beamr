# B-086: term_to_binary encoder — all term types to ETF wire format

> **Cluster:** beamr
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

BEAM's External Term Format (ETF) is the canonical wire encoding for Erlang terms. term_to_binary/1 converts any runtime term to a binary containing the ETF representation. It's used for distribution protocol (node-to-node message passing), persistent storage (ETS dets tables, mnesia), message serialization in ports and NIFs, and phash2 and term comparison across nodes. beamr has a battle-tested ETF decoder in loader/decode/etf.rs (14 tags, budget system, zlib decompression) but no encoder. Without encoding, beamr processes cannot serialize terms for distribution, storage, or cross-boundary communication. This brief adds the encoding direction: given a runtime Term, produce the ETF binary representation. The output is a Vec<u8> that gets allocated as a binary term on the process heap.

## Task

Create the etf/ module with shared tag constants, implement the core encode_term function dispatching on all term types to produce ETF bytes, extend encoding for compound terms (tuples, lists, maps, binaries, bigints, closures) with depth limiting, and register erlang:term_to_binary/1 as a BIF wired to the encoder.

### R2: Core encoder — encode_term function

THE SYSTEM SHALL provide a pub fn encode_term(term: Term, atom_table: &AtomTable) -> Result<Vec<u8>, EncodeError> in etf/encode.rs. WHEN encoding a term, THE SYSTEM SHALL emit a version byte (131) followed by the most compact ETF tag for each value: SMALL_INTEGER_EXT for 0-255, INTEGER_EXT for i32 range, SMALL_BIG_EXT or LARGE_BIG_EXT for outside i32, NEW_FLOAT_EXT for floats (IEEE 754), SMALL_ATOM_UTF8_EXT for atom names <= 255 bytes else ATOM_UTF8_EXT (resolved via atom_table.resolve), NIL_EXT for NIL, NEW_PID_EXT for PIDs with node atom nonode@nohost and creation 0, NEWER_REFERENCE_EXT for references with node atom and creation 0. THE SYSTEM SHALL define an EncodeError type with variants UnsupportedTerm and AtomResolveFailed.

**Acceptance:**
- etf/encode.rs with pub fn encode_term(term: Term, atom_table: &AtomTable) -> Result<Vec<u8>, EncodeError>
- Output begins with version byte (131)
- Small integer (0-255): SMALL_INTEGER_EXT (1+1 bytes)
- Integer (i32 range): INTEGER_EXT (1+4 bytes)
- Large integer (outside i32): SMALL_BIG_EXT or LARGE_BIG_EXT
- Float: NEW_FLOAT_EXT (1+8 bytes, IEEE 754)
- Atom: SMALL_ATOM_UTF8_EXT if name <= 255 bytes, else ATOM_UTF8_EXT. Resolves atom name via atom_table.resolve(atom)
- NIL: NIL_EXT (1 byte)
- PID: NEW_PID_EXT with node atom nonode@nohost, creation 0
- Reference: NEWER_REFERENCE_EXT with node atom, creation 0
- New error type: EncodeError with variants UnsupportedTerm, AtomResolveFailed
- Test: encode small integer 42 -> [131, 97, 42]
- Test: encode integer 1000 -> [131, 98, 0, 0, 3, 232]
- Test: encode float 3.14 -> [131, 70, <8 IEEE bytes>]
- Test: encode atom 'ok' -> [131, 119, 2, 111, 107]
- Test: encode NIL -> [131, 106]

### R3: Compound term encoding — tuples, lists, maps, binaries

WHEN encoding a tuple, THE SYSTEM SHALL use SMALL_TUPLE_EXT for arity <= 255 and LARGE_TUPLE_EXT for larger, recursively encoding each element. WHEN encoding a list, THE SYSTEM SHALL walk the cons chain; if all elements are bytes (0-255) and tail is NIL, THE SYSTEM SHALL encode as STRING_EXT; otherwise THE SYSTEM SHALL encode as LIST_EXT with recursive elements and explicit tail encoding. WHEN encoding a binary, THE SYSTEM SHALL use BINARY_EXT with 4-byte length plus raw bytes. WHEN encoding a map, THE SYSTEM SHALL use MAP_EXT with 4-byte size plus recursively encoded key-value pairs. WHEN encoding a BigInt, THE SYSTEM SHALL use SMALL_BIG_EXT or LARGE_BIG_EXT with sign byte plus little-endian limbs. WHEN encoding an exported closure, THE SYSTEM SHALL use EXPORT_EXT; for captured closures, THE SYSTEM SHALL return EncodeError::UnsupportedTerm. WHEN recursive encoding exceeds 256 nesting levels, THE SYSTEM SHALL return EncodeError::TooDeep.

**Acceptance:**
- Tuple: SMALL_TUPLE_EXT for arity <= 255, LARGE_TUPLE_EXT for larger. Recursively encode each element.
- List (cons): Walk the cons chain. If all elements are bytes (0-255) and tail is NIL, encode as STRING_EXT (optimization). Otherwise, encode as LIST_EXT with recursive elements and explicit tail encoding.
- Binary: BINARY_EXT — 4-byte length + raw bytes. Read bytes via existing Binary::as_bytes().
- Map: MAP_EXT — 4-byte size + key-value pairs. Keys and values recursively encoded.
- BigInt: SMALL_BIG_EXT or LARGE_BIG_EXT — sign byte + little-endian limbs. Read from the existing BigInt heap layout.
- Closure: EXPORT_EXT for exported funs (module:function/arity). For captured closures, return EncodeError::UnsupportedTerm.
- Depth limit: recursive encoding limited to 256 nesting levels (matching the decode budget's MAX_ETF_DEPTH). Exceeding returns EncodeError::TooDeep.
- Test: encode {ok, 42} -> SMALL_TUPLE_EXT with ATOM + SMALL_INTEGER
- Test: encode [1, 2, 3] -> LIST_EXT with 3 integers + NIL tail
- Test: encode byte list [72, 101, 108, 108, 111] -> STRING_EXT optimization
- Test: encode #{a => 1} -> MAP_EXT
- Test: encode binary <<1,2,3>> -> BINARY_EXT
- Test: deeply nested term (>256 levels) returns TooDeep

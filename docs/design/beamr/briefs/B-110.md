# B-110: inet BIF surface — setopts, getopts, peername, sockname

> **Cluster:** beamr
> **Depends on:** B-107, B-109

## Purpose

OTP's inet module provides socket introspection and option management used by both TCP and UDP code. Functions like inet:setopts, inet:getopts, inet:peername, and inet:sockname are called by gen_tcp, gen_udp, and application code. This brief provides the BIF surface for these common socket operations, working with the FdResource type from B-102.

## Task

Implement inet-family BIFs for socket option management and address introspection. These operate on any FdResource-backed socket (TCP or UDP) using standard socket syscalls (getsockopt, setsockopt, getpeername, getsockname).

## Requirements

### R1: inet_setopts and inet_getopts BIFs

THE SYSTEM SHALL provide BIFs for setting and getting socket options on FdResource-backed sockets.

**Acceptance:**
- erlang:inet_setopts/2 with args (Socket, [{Option, Value}])
- erlang:inet_getopts/2 with args (Socket, [Option])
- Supported options: {nodelay, Bool} (TCP_NODELAY), {keepalive, Bool} (SO_KEEPALIVE), {reuseaddr, Bool} (SO_REUSEADDR), {sndbuf, Size} (SO_SNDBUF), {recbuf, Size} (SO_RCVBUF), {active, true|false|once}
- setopts returns ok or {error, Reason}
- getopts returns {ok, [{Option, Value}]} or {error, Reason}
- active option delegates to the mode-switching logic from B-108
- Test: set nodelay, get nodelay, verify value matches

**Files:**
- create: crates/beamr/src/native/inet_bifs.rs
- modify: crates/beamr/src/native/mod.rs

### R2: peername and sockname BIFs

THE SYSTEM SHALL provide BIFs for querying the remote and local addresses of a socket.

**Acceptance:**
- erlang:inet_peername/1 returns {ok, {Address, Port}} or {error, Reason}
- erlang:inet_sockname/1 returns {ok, {Address, Port}} or {error, Reason}
- Address is a 4-tuple of integers for IPv4: {A, B, C, D}
- Port is an integer
- peername on unconnected socket (UDP, listener) returns {error, enotconn}
- Test: connect TCP, verify peername matches listener address
- Test: verify sockname matches bound address

**Files:**
- modify: crates/beamr/src/native/inet_bifs.rs

### R3: inet_port and inet_close BIFs

THE SYSTEM SHALL provide convenience BIFs for getting the port number and closing any socket type.

**Acceptance:**
- erlang:inet_port/1 returns {ok, Port} — shorthand for sockname extracting just the port
- erlang:inet_close/1 delegates to the FdResource close mechanism (B-102 R2)
- Works for both TCP and UDP sockets
- Test: open socket, inet_port returns bound port, inet_close closes it

**Files:**
- modify: crates/beamr/src/native/inet_bifs.rs

## Boundaries

- SHALL NOT implement IPv6 address formatting — IPv4 4-tuples only
- SHALL NOT implement raw socket options (SO_LINGER, IP_TTL) — add as needed
- SHALL NOT implement inet:getstat (socket statistics) — future enhancement

## Verification

- cargo test passes with setopts/getopts round-trip
- peername returns correct remote address after TCP connect
- sockname returns correct local address after bind
- Options set on TCP socket are preserved across active/passive mode switches

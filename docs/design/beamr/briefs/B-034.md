# B-034: GC integration — wire generational copying collector to interpreter loop

> **Cluster:** beamr
> **Depends on:** B-021 (interpreter opcode families)
> **Checklist:**
> - C80 — Per-process heap with generational copying GC

## Purpose

beamr's per-process heap is a fixed-size arena. When a process exhausts its heap, `test_heap` returns `GcNeeded` and the process crashes. The generational copying GC (young/old, Fibonacci growth/shrink) is implemented but not wired to the interpreter's allocation path. Without GC, any Gleam workflow that creates enough intermediate terms (string concatenation loops, list building, map operations) will crash. This is the primary blocker for long-running workflows.

## Task

Wire the existing GC implementation to the interpreter's heap allocation path so that `test_heap` triggers collection instead of crashing when heap space is low.

## Boundaries

- This brief SHALL NOT implement a new GC algorithm — the existing generational copying collector is used
- This brief SHALL NOT implement fullsweep GC — young generation collection is sufficient for Gate 4
- This brief SHALL NOT change the process heap API beyond adding a `collect` entry point
- This brief SHALL ensure GC does not move terms referenced by active x-registers or y-registers

## Requirements

### R1: GC trigger on test_heap failure

When `test_heap` detects insufficient heap space, invoke the garbage collector instead of returning `GcNeeded`. After collection, re-check available space. If still insufficient after GC, grow the heap (Fibonacci growth policy) and retry. Only return an error if growth would exceed a configurable maximum heap size.

**Acceptance:**
- `test_heap` no longer returns GcNeeded for recoverable heap pressure
- A process that creates 10,000 intermediate terms (via a loop) survives without crashing
- Heap size grows as needed via Fibonacci policy
- Maximum heap size is configurable (default: 1MB / 131072 words)

**Files:**
- modify: crates/beamr/src/interpreter/opcodes/core.rs (test_heap function)
- modify: crates/beamr/src/process/heap.rs (add collect entry point)

**Checklist:** C80

### R2: Root set construction

The GC needs a root set — all live term references that must be preserved during collection. The root set consists of:
- x-registers (x0..x255, only the first `live` are roots — `test_heap`'s live parameter)
- y-registers (all slots in all stack frames)
- The process mailbox (all queued messages)
- The current exception term (if any)

Build a root set iterator that yields all live Term values from these sources.

**Acceptance:**
- Root set includes x-registers up to the live count
- Root set includes all y-register values across all stack frames
- Root set includes mailbox messages
- Root set includes current exception reason and stacktrace
- Tests verify root set completeness

**Files:**
- create: crates/beamr/src/process/gc.rs
- modify: crates/beamr/src/process/mod.rs

**Checklist:** C80

### R3: Young generation collection

Implement the young generation copy: scan the root set, copy live objects from young space to old space (or to a new young space if old space is full), update all pointers in the root set and in copied objects.

**Acceptance:**
- Young collection reclaims unreferenced terms
- Referenced terms survive collection with correct values
- Boxed terms (tuples, closures, binaries, maps) are correctly copied
- Cons cells and their tails are correctly traced
- After collection, available heap space increases
- Tests verify: create terms, drop references, collect, verify reclaimed space

**Files:**
- modify: crates/beamr/src/process/gc.rs
- modify: crates/beamr/src/process/heap.rs

**Checklist:** C80

### R4: Integration test — GC under load

End-to-end test that runs a Gleam function which allocates heavily (e.g., building a list of 10,000 integers via recursion), triggers multiple GC cycles, and returns the correct result.

**Acceptance:**
- Process survives 10,000+ allocations
- Return value is correct after multiple GC cycles
- No heap corruption detected by term validation
- Test completes in under 5 seconds

**Files:**
- create: crates/beamr/tests/gc_integration.rs

**Checklist:** C80

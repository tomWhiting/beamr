# B-091: ETS set table type — hash-based storage

> **Cluster:** beamr
> **Depends on:** B-090
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

The `set` table type is the default and most common ETS table. It stores tuples keyed by a specified element position (default: first element). Keys are unique — inserting a tuple with an existing key overwrites the previous entry. Lookup is O(1) via hashing. This is the foundation that bag, duplicate_bag, and most ETS BIFs build on.

## Task

Implement `EtsSet` as a concrete implementation of the `EtsTable` trait using a `DashMap` for concurrent hash-based storage.

## Requirements

### R1: EtsSet implementation

Implement `EtsSet` as a struct wrapping `DashMap<Term, Term>` (key → full tuple) with the EtsTable trait.

**Acceptance:**
- `EtsSet::new(metadata: EtsTableMetadata) -> EtsSet`
- `insert(tuple)`: extracts key at `keypos`, inserts/overwrites in DashMap
- `lookup(key)`: returns vec of 0 or 1 matching tuples (set = unique keys)
- `delete_key(key)`: removes entry, returns true if existed
- `tab2list()`: returns all tuples (unordered)
- Key extraction uses `keypos` from metadata (1-based → 0-based index into tuple elements)
- Non-tuple insert returns EtsError::Badarg
- Test: insert {a, 1}, lookup a → [{a, 1}]. Insert {a, 2}, lookup a → [{a, 2}] (overwritten)

**Files:**
- create: crates/beamr/src/ets/set.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R2: Term hashing for ETS keys

ETS keys are arbitrary terms. Implement term hashing consistent with term equality. Create an EtsKey(Term) newtype that implements Hash (using term_hash) and Eq (using term comparison). DashMap<EtsKey, StoredTerm> uses this newtype for key operations. Do NOT implement Hash directly on Term (global change would affect non-ETS code).

**Acceptance:**
- `term_hash(term: Term) -> u64` produces a deterministic hash
- Atoms hash by atom index
- Integers hash by value
- Binaries hash by content
- Tuples hash by recursive element hashing
- Lists hash by recursive element hashing
- Equal terms (per term comparison) produce equal hashes
- Test: term_hash(atom(ok)) == term_hash(atom(ok))
- Test: term_hash(42) == term_hash(42)
- Test: different terms produce different hashes (probabilistic, test a variety)
- EtsKey newtype wraps Term and implements Hash + Eq
- DashMap uses EtsKey as key type, not raw Term
- EtsKey hashing matches EtsKey equality (Hash contract satisfied)

**Files:**
- create: crates/beamr/src/term/hash.rs
- modify: crates/beamr/src/term/mod.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R3: EtsSet registered as table type

Wire EtsSet into the table creation path so `ets:new(Name, [set])` produces an EtsSet.

**Acceptance:**
- Table creation with `EtsTableType::Set` instantiates an `EtsSet`
- The EtsSet is wrapped in `Arc<dyn EtsTable>` and stored in the registry
- Test: create set table, insert, lookup — round-trips correctly through the registry

**Files:**
- modify: crates/beamr/src/ets/mod.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

# B-104: File seek, pread, pwrite — positional IO

> **Cluster:** beamr
> **Depends on:** B-103

## Purpose

B-103 provides basic file read/write that operates at the current file position. Real file operations often need random access: seeking to a position, reading at a specific offset without moving the position (pread), or writing at a specific offset (pwrite). These are fundamental for database files, log rotation, and any non-sequential file access. io_uring supports pread/pwrite natively as the offset parameter on read/write SQEs, making positional IO zero-cost compared to seek+read.

## Task

Implement file seek, pread, and pwrite BIFs using the CompletionRing. Add file position tracking to FdResource for seek-based operations.

## Requirements

### R1: File position tracking on FdResource

THE SYSTEM SHALL add a current_offset: AtomicU64 field to FdInner for tracking the file position across sequential read/write operations.

**Acceptance:**
- FdInner gains current_offset: AtomicU64, initialized to 0 on open
- Sequential reads advance current_offset by bytes read
- Sequential writes advance current_offset by bytes written
- B-103's read/write BIFs updated to use and advance current_offset

**Files:**
- modify: crates/beamr/src/io/resource.rs
- modify: crates/beamr/src/native/file_bifs.rs

### R2: file_seek BIF

WHEN erlang:file_seek/3 is called with an FdResource, offset, and whence, THE SYSTEM SHALL update the file position and return {ok, NewPosition} or {error, Reason}.

**Acceptance:**
- erlang:file_seek/3 registered as BIF with args (Fd, Offset :: integer(), Whence :: bof | cur | eof)
- Whence atoms: bof (beginning of file), cur (current position), eof (end of file)
- For bof: set current_offset to Offset
- For cur: set current_offset to current_offset + Offset
- For eof: submit statx to get file size, set current_offset to size + Offset
- Returns {ok, NewPosition} on success
- Negative resulting position returns {error, einval}
- Test: seek to position 10, read, verify correct data

**Files:**
- modify: crates/beamr/src/native/file_bifs.rs

### R3: pread and pwrite BIFs

WHEN erlang:pread/3 or erlang:pwrite/3 is called, THE SYSTEM SHALL submit a read/write with the specified offset without changing the FdResource's current_offset.

**Acceptance:**
- erlang:pread/3 registered as BIF with args (Fd, Offset :: non_neg_integer(), Count :: non_neg_integer())
- erlang:pwrite/3 registered as BIF with args (Fd, Offset :: non_neg_integer(), Data :: binary())
- pread submits Read with explicit offset, does NOT update current_offset
- pwrite submits Write with explicit offset, does NOT update current_offset
- Return values match file_read/file_write (B-103)
- Test: write at offset 0, pwrite at offset 100, read at offset 0, verify original data unchanged
- Test: pread at offset 100, verify data from pwrite

**Files:**
- modify: crates/beamr/src/native/file_bifs.rs

## Boundaries

- SHALL NOT implement file truncate — that can be a future addition
- SHALL NOT implement file locking (flock/fcntl) — separate concern
- SHALL NOT change the CompletionRing trait — positional IO uses the existing Read/Write ops with offset parameter

## Verification

- cargo test passes with seek + read verifying correct data at position
- pread/pwrite do not affect the sequential file position
- Seek to EOF returns correct file size
- Negative seek position returns error

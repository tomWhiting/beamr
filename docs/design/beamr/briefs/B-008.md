## Purpose

The native boundary is the point of the whole project — when the runtime IS Rust, native calls are zero-cost. This brief delivers the BIF and NIF registries (the lookup tables that map MFA triples to Rust function pointers), the native function calling convention (Term slice in, Result<Term, Term> out), and the minimum set of BIFs needed for Gate 1 (arithmetic, comparison, error, display). It also implements the concrete BifRegistry that satisfies the trait B-004 defined.

## Task

Implement the BIF and NIF registries in crates/beamr/src/native/. The BifRegistry struct (implementing the BifRegistry trait from B-004) maps (Module, Function, Arity) to a Rust function pointer with the native calling convention: fn(&[Term], &mut Process) -> Result<Term, Term>. The NIF registry uses the same mechanism but is populated by the host application, not built-in. Registration must happen before module loading so BIFs are available for import resolution. Implement the Gate 1 minimum BIFs: erlang:+/2, -/2, */2, div/2, rem/2, </2, >=/2, =:=/2, =/=/2, error/1, display/1. Mark dirty-scheduler support in the registry for future use but do not implement the dirty dispatch yet (scheduler brief scope).

## Requirements

### R1: BIF registry struct implementing BifRegistry trait

beamr::native SHALL define a BifRegistryImpl struct implementing the BifRegistry trait (from B-004). It SHALL store registered BIFs in a map keyed by (Atom, Atom, u8) — module, function, arity. BifRegistryImpl::register(module, function, arity, func) SHALL add a native function. BifRegistryImpl::lookup(module, function, arity) SHALL return the function pointer if registered. It SHALL NOT allow registering the same MFA twice — the second registration SHALL return an error.

**Acceptance:**
- BifRegistryImpl implements the BifRegistry trait from B-004
- register(erlang, +, 2, add_fn) succeeds
- lookup(erlang, +, 2) returns Some(NativeEntry) where entry.function is add_fn and entry.is_dirty is false
- lookup(erlang, unknown, 0) returns None
- Registering the same MFA twice returns an error

### R2: NIF registry

beamr::native SHALL define a NifRegistry struct with the same interface as BifRegistryImpl. NIFs are registered by the host application (Meridian, CLI) rather than built into the VM. The NIF registry SHALL be separate from the BIF registry so that host-registered functions don't collide with built-ins. Import resolution SHALL check BIFs first, then NIFs.

**Acceptance:**
- NifRegistry has register and lookup methods matching BifRegistryImpl
- NifRegistry is separate from BifRegistryImpl — they can be queried independently

### R3: Native calling convention

Native functions SHALL have the signature fn(&[Term], &mut ProcessContext) -> Result<Term, Term> where the first argument is a slice of Term values (the function arguments) and ProcessContext provides access to the calling process's heap for allocating return values. The Ok value is the return term. The Err value becomes the process exit reason. It SHALL NOT expose the full Process struct to native functions — only the ProcessContext subset needed for allocation.

**Acceptance:**
- The NativeFn type alias is defined as fn(&[Term], &mut ProcessContext) -> Result<Term, Term>
- ProcessContext provides a method to allocate terms on the calling process's heap
- A native function returning Ok(Term::small_int(42)) produces 42 as the call result
- A native function returning Err(Term::atom(Atom::BADARITH)) causes the calling process to exit with reason badarith

### R4: Dirty scheduler flag

The registry SHALL support marking a native function as dirty (requiring execution on the dirty scheduler pool). BifRegistryImpl::register_dirty(module, function, arity, func) SHALL set the dirty flag. The lookup result SHALL include the dirty flag so the interpreter knows whether to dispatch to the dirty pool. This brief SHALL NOT implement the dirty dispatch itself — only the flag in the registry. NOTE: C155 is split — this brief delivers the dirty flag on NativeEntry; dirty scheduler dispatch is deferred to the scheduler brief (B-019 or equivalent).

**Acceptance:**
- register_dirty sets a dirty flag on the registration entry
- lookup returns a NativeEntry struct with a is_dirty: bool field
- A non-dirty registration has is_dirty == false
- A dirty registration has is_dirty == true

### R5: Demand-driven BIF scope

The set of implemented BIFs SHALL be driven by the loader's unresolved-import report, not hardcoded to a list. The BIF registry SHALL provide a method to check which registered BIFs cover a given unresolved-import report. It SHALL NOT implement BIFs that no Gleam workflow currently needs — the import report is the leash.

**Acceptance:**
- BifRegistryImpl::coverage(report) returns a list of still-unresolved imports after checking the registry
- If all imports in a report are registered, coverage returns an empty list

### R6: Gate 1 arithmetic BIFs

beamr::native::bifs SHALL implement: erlang:+/2, erlang:-/2, erlang:*/2, erlang:div/2, erlang:rem/2. Each SHALL operate on small integer terms. WHEN both arguments are small integers THE SYSTEM SHALL return a small integer result. IF either argument is not an integer THEN THE SYSTEM SHALL return Err(Atom::BADARITH as Term). IF the result overflows the small integer range THE SYSTEM SHALL return Err(Atom::BADARITH as Term) for now (big integer support is deferred).

**Acceptance:**
- erlang:+/2 with args [3, 4] returns Ok(Term::small_int(7))
- erlang:-/2 with args [10, 3] returns Ok(Term::small_int(7))
- erlang:*/2 with args [3, 4] returns Ok(Term::small_int(12))
- erlang:div/2 with args [7, 2] returns Ok(Term::small_int(3))
- erlang:rem/2 with args [7, 2] returns Ok(Term::small_int(1))
- erlang:div/2 with args [7, 0] returns Err (division by zero → badarith)
- erlang:+/2 with args [atom, 1] returns Err (badarith)

### R7: Gate 1 comparison and utility BIFs

beamr::native::bifs SHALL implement: erlang:</2, erlang:>=/2, erlang:=:=/2, erlang:=/=/2, erlang:error/1, erlang:display/1. Comparison BIFs SHALL return the atom true or false as a Term. erlang:error/1 SHALL return Err with the argument as the exit reason. erlang:display/1 SHALL print the term to stdout (using Debug formatting) and return the atom true. It SHALL NOT implement full term Display formatting — Debug is sufficient for Gate 1.

**Acceptance:**

- erlang:=:=/2 with args [1, 2] returns Ok(Term::atom(Atom::FALSE))
- erlang:error/1 with arg Term::atom(Atom::BADARG) returns Err(Term::atom(Atom::BADARG))
- erlang:display/1 prints to stdout and returns Ok(Term::atom(Atom::TRUE))

## Boundaries

- SHALL NOT implement dirty scheduler dispatch — only the dirty flag in the registry (scheduler brief scope)
- SHALL NOT implement float argument handling in arithmetic BIFs — deferred to a post-Gate-1 brief
- SHALL NOT implement float arithmetic in BIFs — Gate 1 needs integer arithmetic only
- SHALL NOT implement big integer arithmetic — overflow returns badarith for now
- SHALL NOT implement process-related BIFs (spawn, self, send) — process brief scope
- SHALL NOT implement type-check BIFs (is_atom, is_integer, etc.) — those are opcodes, not BIF calls

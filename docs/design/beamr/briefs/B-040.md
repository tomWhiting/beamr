# B-040: Gleam stdlib wrappers — second wave and sample workflow e2e

> **Cluster:** beamr
> **Depends on:** B-037 (core erlang BIFs), B-038 (collection BIFs), B-039 (string/IO BIFs)
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

The gleam_stdlib module contains 29 Erlang FFI wrapper functions that Gleam's standard library delegates to. These wrappers call into the erlang, maps, string, and io BIFs implemented in B-037 through B-039. With all dependencies in place, implementing these wrappers unblocks the sample_workflow:run/1 end-to-end test — the milestone that proves beamr can execute a real Gleam workflow.

## Task

Implement 29 gleam_stdlib BIFs, 2 uri_string stubs, and un-ignore the sample_workflow_e2e test to verify end-to-end execution.

## Boundaries

- SHALL NOT implement full RFC 3986 URI parsing — stub uri_string:parse/1 with approximate splitting
- SHALL NOT implement full Erlang format string parsing — gleam_stdlib:float_to_string uses basic f64 formatting
- SHALL stub uri_string functions as approximations, noted in boundaries
- SHALL mark approximate stubs with inline notes for future replacement

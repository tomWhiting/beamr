# B-066: Numeric equality and ordering BIFs

> **Cluster:** beamr
> **Depends on:** (none — builds on landed comparison infrastructure)
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

BEAM comparison operators can be used in two contexts: as guard-test opcodes (opcodes 39–44, already implemented) and as BIF calls (when passed as function references to higher-order functions like `lists:sort/2`).

The opcode path is complete — all six `ComparisonOp` variants (`Lt`, `Ge`, `Eq`, `Ne`, `EqExact`, `NeExact`) decode and dispatch correctly, backed by `compare::numeric_eq()` for `==`/`/=` and `compare::cmp()` for ordering. The semantics are correct: `1 == 1.0` is true (numeric coercion), `1 =:= 1.0` is false (exact), and term ordering follows the BEAM standard `number < atom < reference < fun < port < pid < tuple < map < nil < list < binary`.

However, only four comparison BIFs are registered: `<`, `>=`, `=:=`, `=/=`. The four numeric variants `==`, `/=`, `>`, `=<` are missing. Any code that references these operators as functions — `fun erlang:'=='/2`, `lists:sort(fun erlang:'>'/2, L)`, or any Gleam code compiled to use these BIFs — fails with `undef`.

## Boundaries

- This brief SHALL register four new comparison BIFs
- This brief SHALL NOT modify the opcode comparison handler (already correct)
- This brief SHALL NOT modify `compare::numeric_eq()` or `compare::cmp()` (already correct)
- This brief SHALL NOT address BigInt-float numeric comparison edge cases (pre-existing limitation, separate concern)

## Requirements

### R1: Numeric equality BIFs

Register `==` and `/=` as erlang BIFs using the existing `compare::numeric_eq()` function.

**Acceptance:**
- `erlang:'=='/2` registered in BIF registry with `Capability::Pure`
- Implementation: `compare::numeric_eq(left, right)` → returns `Atom::TRUE` or `Atom::FALSE`
- `erlang:'/='/2` registered in BIF registry with `Capability::Pure`
- Implementation: `!compare::numeric_eq(left, right)` → returns `Atom::TRUE` or `Atom::FALSE`
- Test: `'=='(1, 1.0)` returns true (numeric coercion)
- Test: `'=='(1, 2)` returns false
- Test: `'/='(1, 1.0)` returns false
- Test: `'/='(1, 2)` returns true

**Files:**
- modify: crates/beamr/src/native/bifs.rs (registration + handlers)

**Checklist:** C150

### R2: Ordering BIFs

Register `>` and `=<` as erlang BIFs using the existing `compare::cmp()` function.

**Acceptance:**
- `erlang:'>'/2` registered in BIF registry with `Capability::Pure`
- Implementation: `compare::cmp(left, right, atom_table) == Ordering::Greater` → returns `Atom::TRUE` or `Atom::FALSE`
- `erlang:'=<'/2` registered in BIF registry with `Capability::Pure`
- Implementation: `compare::cmp(left, right, atom_table) != Ordering::Greater` → returns `Atom::TRUE` or `Atom::FALSE`
- BIF handlers need atom table access — use the same pattern as existing `less_than` and `greater_equal` BIFs
- Test: `'>'(2, 1)` returns true
- Test: `'>'(1, 2)` returns false
- Test: `'=<'(1, 2)` returns true
- Test: `'=<'(2, 1)` returns false
- Test: `'>'(1.0, 1)` returns false (equal numerically)
- Test: `'=<'(1.0, 1)` returns true (equal numerically)

**Files:**
- modify: crates/beamr/src/native/bifs.rs (registration + handlers)

**Checklist:** C150

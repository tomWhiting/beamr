# B-035: Term-to-Value bridge — bidirectional serde_json::Value conversion

> **Cluster:** beamr
> **Depends on:** B-033 (binary handling BIFs)
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

The Meridian workflow executor passes inputs as `HashMap<String, serde_json::Value>` and expects outputs as `serde_json::Value`. Gleam functions operate on BEAM terms. A bidirectional converter between `Term` and `serde_json::Value` is needed for the beamr-meridian integration layer. This module belongs in the beamr core crate (not beamr-meridian) because any host program embedding beamr will need JSON interop.

## Task

Implement a `term::json` module with `term_to_value(Term) -> serde_json::Value` and `value_to_term(Value, &mut ProcessContext) -> Term` conversions.

## Boundaries

- This brief SHALL NOT implement custom struct serialization (no derive macros)
- This brief SHALL NOT add serde as a required dependency of beamr — gate behind a `json` feature flag
- This brief SHALL handle all immediate and common boxed term types

## Requirements

### R1: Term to serde_json::Value

Convert a BEAM Term to a `serde_json::Value`:

| Term type | JSON value |
|-----------|------------|
| SmallInt | Value::Number (i64) |
| Atom (true/false) | Value::Bool |
| Atom (nil/undefined) | Value::Null |
| Atom (other) | Value::String (atom name) |
| Nil | Value::Array (empty) |
| Binary (UTF-8) | Value::String |
| Binary (non-UTF-8) | Value::String (base64 encoded) |
| Tuple | Value::Array |
| Cons list | Value::Array (flatten to elements) |
| Map | Value::Object |
| Float | Value::Number (f64) |
| Pid | Value::String (formatted as "<0.N.0>") |
| BigInt | Value::Number or Value::String for overflow |

**Acceptance:**
- `term_to_value(Term::small_int(42))` returns `Value::Number(42)`
- `term_to_value(Term::atom(Atom::TRUE))` returns `Value::Bool(true)`
- `term_to_value(binary_term("hello"))` returns `Value::String("hello")`
- `term_to_value(tuple_term([ok, 42]))` returns `Value::Array([String("ok"), Number(42)])`
- `term_to_value(map_term({key: value}))` returns `Value::Object({"key": value})`
- Nested structures convert recursively
- Tests cover each term type

**Files:**
- create: crates/beamr/src/term/json.rs
- modify: crates/beamr/src/term/mod.rs
- modify: crates/beamr/Cargo.toml (serde_json feature flag)

**Checklist:** C150

### R2: serde_json::Value to Term

Convert a `serde_json::Value` to a BEAM Term:

| JSON value | Term type |
|------------|-----------|
| Value::Null | Term::NIL |
| Value::Bool(true) | Term::atom(Atom::TRUE) |
| Value::Bool(false) | Term::atom(Atom::FALSE) |
| Value::Number (integer) | Term::small_int |
| Value::Number (float) | Float term (heap allocated) |
| Value::String | Binary term (heap allocated) |
| Value::Array | Cons list of converted elements |
| Value::Object | Map term (heap allocated) |

Heap allocation uses `ProcessContext::alloc_tuple`, `alloc_cons`, and leaked-heap patterns for binaries. This is acceptable for input conversion (short-lived allocation before the process starts).

**Acceptance:**
- `value_to_term(Value::Number(42), ctx)` returns `Term::small_int(42)`
- `value_to_term(Value::String("hello"), ctx)` returns a binary term
- `value_to_term(Value::Array([1, 2, 3]), ctx)` returns a proper cons list
- `value_to_term(Value::Object({"key": "value"}), ctx)` returns a map term
- Round-trip: `term_to_value(value_to_term(v))` preserves structure for all JSON types
- Tests cover each type and round-trip

**Files:**
- modify: crates/beamr/src/term/json.rs

**Checklist:** C150

### R3: AtomTable integration for string↔atom conversion

`term_to_value` needs the AtomTable to look up atom names. `value_to_term` needs it to intern atom strings. Add an `AtomTable` parameter (or make it available via ProcessContext).

**Acceptance:**
- `term_to_value(atom_term, atom_table)` produces the correct string
- `value_to_term(Value::String("ok"), ctx)` when the result should be an atom (convention: keys in objects become atoms)
- No panics for unknown atom IDs

**Files:**
- modify: crates/beamr/src/term/json.rs

**Checklist:** C150

# B-115: Distribution handshake protocol — challenge-response authentication

> **Cluster:** beamr
> **Depends on:** B-114
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

The distribution handshake authenticates connecting nodes using a shared cookie and challenge-response. Both nodes exchange names, capabilities (distribution flags), and prove knowledge of the cookie via MD5 challenge hashing. This prevents unauthorized nodes from joining the cluster.

## Task

Implement the BEAM distribution handshake protocol for both outbound (initiator) and inbound (responder) sides.

## Requirements

### R1: Handshake initiator (outbound)

Implement the client side of the handshake: send name + flags, receive challenge, compute response, send response, receive status.

**Acceptance:**
- Sends node name and distribution flags to remote
- Receives challenge, computes MD5(cookie + challenge) response
- Sends response, receives 'ok' status
- Failed handshake returns HandshakeError
- Test: complete handshake between two local nodes

**Files:**
- create: crates/beamr/src/distribution/handshake.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R2: Handshake responder (inbound)

Implement the server side: receive name, send challenge, receive response, verify, send status.

**Acceptance:**
- Receives remote node name and flags
- Sends challenge to remote
- Verifies response matches MD5(cookie + challenge)
- Sends 'ok' on success, 'not_allowed' on failure
- Test: reject connection with wrong cookie

**Files:**
- modify: crates/beamr/src/distribution/handshake.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R3: Distribution flags negotiation

Exchange and negotiate capability flags during handshake.

**Acceptance:**
- DistributionFlags bitfield with: PUBLISHED, ATOM_CACHE, EXTENDED_REFERENCES, EXTENDED_PIDS, UTF8_ATOMS, BIG_CREATION, MAP_TAG, HANDSHAKE_23
- Both sides send their flags, intersection determines connection capabilities
- Minimum required flags enforced (reject incompatible nodes)
- Test: flag negotiation produces correct intersection

**Files:**
- modify: crates/beamr/src/distribution/handshake.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Boundaries

- SHALL NOT implement TLS — plain TCP with cookie authentication only
- SHALL NOT implement distribution flag FRAGMENTS (large message fragmentation) — that's an optimization for later
- SHALL implement the OTP 23+ handshake (version 6), not the legacy handshake

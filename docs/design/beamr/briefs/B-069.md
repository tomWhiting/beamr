# B-069: Tail-call BIF return fix — call_ext_only with Native target

> **Cluster:** beamr
> **Depends on:** B-013 (external calls and import resolution)
> **Checklist:**
> - C65 — call_ext_only jumps to external target without saving return address (tail call)

## Purpose

`call_ext_only` is a tail call — the current function's frame is already deallocated and the callee's result IS the caller's result. When the target is a BEAM function, this works correctly (the IP jumps and never returns). But when the target is a Native BIF, `call_external_target()` executes the BIF, stores the result in x(0), and returns `InstructionOutcome::Continue` — which falls through to the next instruction in the code stream. This is wrong: there IS no next instruction after a tail call. The code that happens to follow is the beginning of the next function in the module.

This bug is masked by flat code layouts where the next function starts with `func_info` (which raises a function_clause error, killing the process "normally"). But it causes silent corruption when code layout differs, and it always wastes cycles executing dead code.

## Task

Fix `call_external_target()` to handle the tail-call case for Native BIF targets. When `save_return` is false (tail call), after executing the BIF, perform a return instead of continuing.

## Boundaries

- SHALL NOT change the non-tail-call path — `call_ext` with `save_return=true` correctly pushes a return address and continues
- SHALL NOT change the BEAM function (Code target) path — that already handles tail calls correctly
- SHALL NOT change the `NativeFn` signature or return type
- SHALL fix all three tail-call variants: `call_ext_only`, `call_ext_last`, and any apply variant that is a tail call

## Requirements

### R1: Detect tail-call context in Native BIF dispatch

The `call_external_target()` function receives a `save_return: bool` parameter. When `save_return` is false and the target is `ResolvedImportTarget::Native`, the BIF result must be returned to the caller, not continued past.

**Acceptance:**
- When `save_return=false` and target is Native: after executing the BIF and storing result in x(0), perform a return (pop the return address from the stack and jump to it)
- When `save_return=true` and target is Native: existing behavior unchanged (Continue)
- The return uses the same mechanism as the `return` opcode (pop frame, jump to return IP)
- If the stack is empty (initial call frame), return `Exited(Normal)` — the BIF result was the process's final value
- Test: `call_ext_only` to a BIF that returns 42 — x(0) is 42 and execution returns to the caller, not falls through

**Files:**
- modify: crates/beamr/src/interpreter/opcodes/core.rs (call_external_target, around lines 318-399)

**Checklist:** C65

### R2: call_ext_last with Native BIF

`call_ext_last` deallocates N stack slots then jumps (tail call with deallocation). Verify this variant also handles Native BIFs correctly after the R1 fix.

**Acceptance:**
- `call_ext_last` with a Native BIF target: deallocates the frame, executes the BIF, stores result in x(0), returns to caller
- Stack frame is deallocated BEFORE the BIF executes (matching BEAM semantics — the BIF runs in the caller's context)
- Test: function with local variables calls a BIF via `call_ext_last` — frame is deallocated, BIF result returned correctly

**Files:**
- modify: crates/beamr/src/interpreter/opcodes/core.rs

**Checklist:** C65

### R3: Regression test with known-broken pattern

Create a test that specifically exercises the bug pattern: a function whose last instruction is `call_ext_only` to a BIF, followed by a different function in the same module.

**Acceptance:**
- Test: module with `foo() -> erlang:'+' (1, 2)` (compiles to call_ext_only to +/2) followed by `bar() -> 99`. Call foo() — returns 3, not 99 or function_clause error.
- Test: module where foo() tail-calls a BIF and the process exits normally with the correct return value
- Test: nested tail calls — foo() tail-calls bar() which tail-calls a BIF — correct result propagates

**Files:**
- modify: crates/beamr/src/interpreter/opcodes/core.rs (test section)

**Checklist:** C65

# B-070: recv_marker opcodes 173-176 — OTP 24+ selective receive optimization

> **Cluster:** beamr
> **Depends on:** B-019
> **Checklist:**
> - C83 — loop_rec, loop_rec_end, remove_message instructions: selective receive mailbox scan
> - C127 — Selective receive scans the mailbox from the save pointer forward, testing each message against patterns

## Purpose

OTP 24+ introduced `recv_marker` opcodes (173-176) to optimize selective receive. These mark the mailbox scan position before a receive so subsequent receives can skip already-tested messages. Without these opcodes, any module compiled with OTP 24+ that uses receive fails to load with `unsupported opcode 173`.

The existing mailbox already has `save_pointer: usize` infrastructure from B-019. The recv_marker opcodes provide explicit control over this pointer from compiled code, rather than the implicit management in the current `wait`/`loop_rec` cycle.

## Task

Add opcode arity definitions, loader decode, Instruction variants, and dispatch handlers for opcodes 173-176. For the initial implementation, implement as functional stubs that interact with the existing save pointer — the basic selective receive optimization. Full marker-based optimization (multiple independent markers per process) is deferred.

## Requirements

### R1: Opcode arity and loader decode

Add opcode arity definitions for opcodes 173-176 and decode them into dedicated Instruction variants.

**Acceptance:**
- Opcode 173 (recv_marker_reserve): arity 1, decoded to `Instruction::RecvMarkerReserve { dest }`
- Opcode 174 (recv_marker_bind): arity 2, decoded to `Instruction::RecvMarkerBind { marker, label }`
- Opcode 175 (recv_marker_clear): arity 1, decoded to `Instruction::RecvMarkerClear { marker }`
- Opcode 176 (recv_marker_use): arity 1, decoded to `Instruction::RecvMarkerUse { marker }`
- Modules compiled with OTP 24+ that use receive load without errors
- Test: load a .beam file containing recv_marker instructions — no load error

**Files:**
- modify: crates/beamr/src/loader/decode/opcode.rs (arity table)
- modify: crates/beamr/src/loader/decode/code.rs (decode switch)
- modify: crates/beamr/src/loader/decode/instruction.rs (Instruction enum)

**Checklist:**
- C83 — loop_rec, loop_rec_end, remove_message instructions: selective receive mailbox scan
- C127 — Selective receive scans the mailbox from the save pointer forward, testing each message against patterns

### R2: Dispatch wiring and handlers

Wire all four recv_marker instructions into the dispatch table. Implement handlers that interact with the mailbox save pointer.

**Acceptance:**
- `recv_marker_reserve`: stores current mailbox length as the marker value in the destination register (or sets save pointer)
- `recv_marker_bind`: binds the marker to the receive label — for the stub, this is a no-op (the save pointer already tracks position)
- `recv_marker_clear`: resets the save pointer to 0 (clear the optimization hint)
- `recv_marker_use`: sets the save pointer to the marker value (skip already-scanned messages)
- All four dispatch without `UnsupportedOpcode` errors
- Existing selective receive tests continue to pass

**Files:**
- modify: crates/beamr/src/interpreter/opcodes/mod.rs (dispatch table)
- modify: crates/beamr/src/interpreter/opcodes/messaging.rs (handlers — 685 lines, over 500; see R3)

**Checklist:**

# B-113: Name resolution service — pluggable resolver replacing EPMD

> **Cluster:** beamr
> **Depends on:** B-112
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

BEAM nodes find each other via EPMD (Erlang Port Mapper Daemon), a standalone process that maps node names to TCP ports on a host. beamr replaces EPMD with a pluggable resolver trait — the default implementation uses DNS SRV records or a static config file, but operators can plug in service discovery (Consul, Kubernetes DNS, etc.).

## Task

Define the NodeResolver trait and a static resolver implementation. The resolver maps node names to (host, port) pairs.

## Requirements

### R1: NodeResolver trait

Define an async trait for resolving node names to network addresses.

**Acceptance:**
- NodeResolver trait with async fn resolve(name: &str) -> Result<SocketAddr, ResolveError>
- ResolveError enum: NotFound, NetworkError(String)
- Resolver is Arc<dyn NodeResolver + Send + Sync> for shared use
- Test: mock resolver returns configured address

**Files:**
- create: crates/beamr/src/distribution/resolver.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R2: Static resolver implementation

A resolver backed by a static map of node names to addresses, loaded from configuration.

**Acceptance:**
- StaticResolver::new(map: HashMap<String, SocketAddr>) creates a resolver
- Resolves known names to configured addresses
- Returns NotFound for unknown names
- Test: configure 3 nodes, resolve each, verify addresses

**Files:**
- modify: crates/beamr/src/distribution/resolver.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R3: Resolver registration in distribution config

The distribution subsystem holds a configured resolver, defaulting to StaticResolver with no entries (non-distributed mode).

**Acceptance:**
- DistributionConfig struct with resolver: Arc<dyn NodeResolver>
- Default config uses empty StaticResolver
- Scheduler gains optional DistributionConfig
- Test: default config resolves nothing (non-distributed mode)

**Files:**
- modify: crates/beamr/src/distribution/mod.rs
- modify: crates/beamr/src/scheduler/mod.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Boundaries

- SHALL NOT implement EPMD compatibility — beamr uses a pluggable resolver, not EPMD
- SHALL NOT implement DNS SRV resolution in this brief — that's a future resolver implementation
- SHALL NOT open network connections — that's B-114

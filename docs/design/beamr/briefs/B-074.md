# B-074: Sub-binary term type — offset+length views into parent binaries

> **Cluster:** beamr
> **Depends on:** B-072 (SharedBinary and ProcBin)
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

When BEAM pattern-matches a binary (`<<Prefix:3/binary, Rest/binary>>`), it currently creates a full copy of `Rest` as a new inline binary. For a 1MB binary split at byte 3, this copies 999,997 bytes. A sub-binary is a lightweight view: it stores an offset and length into a parent binary without copying data. Combined with refc binaries (B-072), extracting a sub-binary from a ProcBin creates a 5-word heap term instead of copying potentially megabytes.

## Task

Add a SubBinary on-heap term type (BoxedTag 0x1A) that references a parent binary (inline or ProcBin) with an offset and length. Modify binary pattern matching to produce sub-binaries instead of copies for refc parents.

## Boundaries

- SHALL NOT implement writable sub-binaries — sub-binaries are read-only views
- SHALL NOT modify inline binary extraction for small binaries — only refc parent binaries produce sub-binaries
- SHALL NOT implement sub-binary-of-sub-binary flattening in this brief — that's an optimization for later

## Requirements

### R1: SubBinary heap layout

Define the SubBinary on-heap term with BoxedTag 0x1A.

**Acceptance:**
- SubBinary heap layout: `[header(tag=0x1A, size=4)] [parent_term: u64] [offset_bytes: u64] [length_bytes: u64] [flags: u64]`
- `write_sub_binary(heap, parent_term, offset, length) -> Option<Term>` writes the SubBinary
- SubBinary is always 5 words regardless of the view size
- Parent term can be either an inline Binary or a ProcBin
- Test: create SubBinary viewing bytes 10..20 of a ProcBin — 5 heap words used

**Files:**
- create: crates/beamr/src/term/sub_binary.rs
- modify: crates/beamr/src/term/boxed/mod.rs (add BoxedTag::SubBinary = 0x1A)
- modify: crates/beamr/src/term/mod.rs

**Checklist:** C150

### R2: SubBinary accessor

Add a `SubBinary` accessor struct that resolves the parent and returns the correct byte slice.

**Acceptance:**
- `SubBinary::new(term: Term) -> Option<Self>` validates BoxedTag::SubBinary
- `SubBinary::as_bytes(self) -> &[u8]` returns `parent.as_bytes()[offset..offset+length]`
- `SubBinary::len(self) -> usize` returns the view length
- `SubBinary::parent(self) -> Term` returns the parent term
- Handles both inline Binary and ProcBin parents
- Test: SubBinary of ProcBin returns correct byte slice

**Files:**
- modify: crates/beamr/src/term/boxed/accessors.rs

**Checklist:** C150

### R3: Update BinaryRef for sub-binaries

Extend the `BinaryRef` accessor (from B-072 R4) to handle SubBinary terms as a third variant.

**Acceptance:**
- `BinaryRef::new(term)` returns `BinaryRef::Sub(SubBinary)` for SubBinary terms
- `BinaryRef::as_bytes()` works transparently for sub-binary views
- All code using BinaryRef automatically supports sub-binaries
- Test: BinaryRef on a SubBinary returns the correct byte slice

**Files:**
- modify: crates/beamr/src/term/binary_ref.rs

**Checklist:** C150

### R4: Binary matching produces sub-binaries for refc parents

Modify `bs_get_binary()` to produce a SubBinary when the match context's source is a ProcBin, instead of copying bytes into a new inline binary.

**Acceptance:**
- `bs_get_binary()` extracting from a ProcBin source creates a SubBinary (5 heap words, no byte copy)
- `bs_get_binary()` extracting from an inline Binary continues to create an inline copy (existing behavior)
- The extracted sub-binary is readable via BinaryRef
- Test: match 10 bytes from a 1MB ProcBin — SubBinary created, parent's ref count unchanged
- Test: match bytes from an inline binary — inline copy created (existing behavior preserved)

**Files:**
- modify: crates/beamr/src/interpreter/opcodes/binary.rs

**Checklist:** C150

### R5: GC and object_size for SubBinary

Add SubBinary to the GC's object_size and rewrite functions. SubBinary contains a parent Term pointer that must be traced during collection.

**Acceptance:**
- `object_size()` returns 5 for BoxedTag::SubBinary
- `rewrite_copied_object()` traces the parent_term field (word at offset 1) — this IS a Term pointer that may need updating
- After GC, the SubBinary's parent pointer correctly references the moved parent
- Test: SubBinary with ProcBin parent survives GC with correct byte access

**Files:**
- modify: crates/beamr/src/gc/mod.rs

**Checklist:** C150

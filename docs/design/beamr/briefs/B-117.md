# B-117: Distribution message encoding — ETF wire format

> **Cluster:** beamr
> **Depends on:** B-116
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

Distribution messages are encoded using the External Term Format (ETF) — the same binary format used by term_to_binary/binary_to_term. Every term type (atoms, integers, tuples, lists, maps, PIDs, references, binaries) has a defined wire encoding. This brief implements the encoder and decoder for the full term type catalog.

## Task

Implement ETF encoding (Term → bytes) and decoding (bytes → Term) for all term types used in distribution.

## Requirements

### R1: ETF encoder — Term to bytes

Encode a Term to its ETF binary representation.

**Acceptance:**
- encode_term(term: Term, atom_table: &AtomTable) -> Vec<u8>
- Encodes: SMALL_INTEGER_EXT, INTEGER_EXT, ATOM_UTF8_EXT, SMALL_TUPLE_EXT, LARGE_TUPLE_EXT, NIL_EXT, LIST_EXT, BINARY_EXT, NEW_FLOAT_EXT, MAP_EXT, NEW_PID_EXT, NEWER_REFERENCE_EXT
- Output starts with version byte 131
- Test: encode each term type, verify byte output matches BEAM's encoding

**Files:**
- create: crates/beamr/src/distribution/etf.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R2: ETF decoder — bytes to Term

Decode an ETF binary representation into a Term, allocating boxed terms on a provided heap.

**Acceptance:**
- decode_term(bytes: &[u8], heap: &mut Heap, atom_table: &AtomTable) -> Result<Term, DecodeError>
- Handles all tags from R1
- Heap allocation for boxed terms (tuples, lists, maps, binaries)
- Invalid input returns DecodeError, not panic
- Test: round-trip encode/decode for each term type

**Files:**
- modify: crates/beamr/src/distribution/etf.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R3: Distribution message framing

Wrap encoded terms in the distribution message frame: 4-byte length prefix + pass-through header + control message + optional payload.

**Acceptance:**
- write_dist_message(control: &[u8], payload: Option<&[u8]>) -> Vec<u8> produces a framed message
- read_dist_message(stream) -> Result<(Vec<u8>, Option<Vec<u8>>), Error> reads and deframes
- Length prefix is big-endian u32
- Test: frame a message, deframe it, verify content

**Files:**
- modify: crates/beamr/src/distribution/etf.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Boundaries

- SHALL NOT implement COMPRESSED_EXT (zlib compression) — optimization for later
- SHALL NOT implement ATOM_CACHE_REF encoding here — that's B-116's responsibility
- SHALL implement the NEW_PID_EXT and NEWER_REFERENCE_EXT formats (not legacy V4_PORT_EXT)

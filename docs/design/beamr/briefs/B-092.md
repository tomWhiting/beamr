# B-092: ETS bag and duplicate_bag table types

> **Cluster:** beamr
> **Depends on:** B-091
> **Checklist:**
> - C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Purpose

`bag` tables allow multiple tuples with the same key but reject exact duplicates. `duplicate_bag` allows fully duplicate tuples. Both are used in OTP for multi-value indexes (e.g., process group membership, event handlers).

## Task

Implement `EtsBag` and `EtsDuplicateBag` as EtsTable implementations using `DashMap<Term, Vec<Term>>` (key → list of tuples).

## Requirements

### R1: EtsBag implementation

EtsBag implementation

**Acceptance:**
- `insert(tuple)`: adds tuple to the key's list. If exact duplicate exists, no-op (bag rejects exact duplicates)
- `lookup(key)`: returns all tuples with that key
- `delete_key(key)`: removes all tuples with that key
- Test: insert {a,1}, insert {a,2} → lookup a returns [{a,1},{a,2}]
- Test: insert {a,1} twice → lookup a returns [{a,1}] (duplicate rejected)

**Files:**
- create: crates/beamr/src/ets/bag.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R2: EtsDuplicateBag implementation

EtsDuplicateBag implementation

**Acceptance:**
- `insert(tuple)`: always adds, even if exact duplicate exists
- `lookup(key)`: returns all tuples including duplicates
- Test: insert {a,1} twice → lookup a returns [{a,1},{a,1}]

**Files:**
- modify: crates/beamr/src/ets/bag.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

### R3: Wire bag types into table creation

Wire bag types into table creation

**Acceptance:**
- `EtsTableType::Bag` instantiates EtsBag
- `EtsTableType::DuplicateBag` instantiates EtsDuplicateBag
- Test: create bag table, insert duplicates, verify correct semantics

**Files:**
- modify: crates/beamr/src/ets/mod.rs

**Checklist:**
- C150 — BIF registry maps (Module, Function, Arity) to a Rust function pointer

## Boundaries

- SHALL NOT implement ordered iteration — bags are unordered like sets
- SHALL NOT implement match specifications — that's B-096/B-097
